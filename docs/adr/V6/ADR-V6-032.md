  - §7.3 line 746：Phase 0 建 `realityos_*` 骨架 + 「Quark 接口 IDL（即使 stub）」
  - §10 line 798：「QuarkExtractor/TheoryEngine 具体算法（Phase 2 设计，**接口 Phase 0 定**）」
- 但 fork 实代码：**零契约**。亲核（Explore 91 tool-use）确认全仓无任何 `class*Quark` / `class*Theory` / `Protocol` / `ABC` / `TypedDict`；唯一痕迹是 `plugins/memory/ptg/schema.py:410` 预留的 `tool_events.quark_evidence TEXT DEFAULT '[]'` 列 + `capture_schemas.py` 的 `CaptureEvent.quark_evidence` placeholder——**只留了列，没钉契约**（什么填它、什么消费它、填什么形状，全无定义）。

母纲决策（action 16）：「Quark/Theory IDL stub（Phase 2 前，**ADR-V6-032**）」。

## 决策

新增 `plugins/memory/ptg/phase2_contracts.py`，把 Phase 2 的接口契约**现在定死**（IDL），**不含任何 Phase 2 实现**：

### D1：契约内容（全部源自设计总纲 §4.3E/§0.9/§13 逐字意图，非发明）

- **Quark 层**：
  - `QUARK_KINDS`：7 个 Meta Quark（删 Exception=R11 重复后；§4.3E line 371）。
  - `PHASE2_QUARK_KINDS = ("Identity","Meaning","Feeling")`：Phase 2 实际只交付这 3 个文本版（§4.3E line 374）；Time/Behavior/Context/Network 钉形状但 Phase 2 不实现（line 373，V6 砍 Layer B/C 连续音频致来源受限，防空跑）。
  - `QUARK_TO_ATOM_MAP`：Quark→原子聚合映射（§4.3E line 377-388「REV-3 补全」表逐条编码：Identity→R3/R1、Meaning→R7/R2/R8、Feeling→R9/R1、Time→R10/R11、Behavior→R6、Context→R5、Network→R3）。
  - `QuarkRecord`（pydantic）：kind/value/source_id/occurrence_count/confidence/evidence。
  - `QuarkExtractor`（`runtime_checkable Protocol`）：`extract(quark_evidence_rows, capture_text) -> list[QuarkRecord]`，消费 `tool_events.quark_evidence`。
- **Theory 层**：
  - `PC_CONSTRAINTS`：7 个 PC 约束（Time/Energy/Cognition/Emotion/Social/Execution/Environment；line 116）。
  - `FR_DIMENSIONS`：5 维 FR（Career/Interpersonal/BodyMind/Learning/Finance；line 116）。
  - `THEORY_DERIVATIONS`：相变/PASCR/守恒（line 116/389）。
  - `TheoryDerivation`（pydantic）：kind/name/score/rationale/aggregation_type(仅 `constraint_state`|`fr_snapshot`，排除 `detected_pattern`——那是 Pattern-Discovery sink 非 Theory 输出；line 288/389)/confidence。
  - `TheoryEngine`（`runtime_checkable Protocol`）：`derive(user_id, atoms, relations) -> list[TheoryDerivation]`，落 `insight_aggregation`，**只读原子层**（§4.7 单向数据流不变式）。
- `PHASE2_CONTRACT_VERSION = 1`：契约任何 Phase-2-可见改动须 bump（C1：契约变更即决策）。

### D2：反假绿保证（契约 only，零 Phase-2 实现）

- 模块**只定义** Protocol（接口形状）+ pydantic BaseModel（数据形状）+ 常量。**无任何具体 extractor/engine 类**。
- 测试 `tests/test_phase2_contracts.py`（28 测）钉死：
  - `test_only_protocols_and_pydantic_models_ship`：模块内每个类**必须**是 Protocol 或 BaseModel 之一——未来若有人加具体实现类，此测红，强制走 ADR + 承认 Phase 2 已启动（防悄悄塞 Phase-2 半成品伪装 done）。
  - 设计保真：7 Quark / Phase-2 子集 3 / Quark→atom 映射 7 条 / 7 PC / 5 FR 全逐条对账 §4.3E。
  - Protocol 结构化可 runtime-check；pydantic 边界（confidence 0-1、aggregation_type 枚举、kind 枚举）生效。

### D3：落点 = 单文件，不建 dormant 插件目录

契约放 `plugins/memory/ptg/phase2_contracts.py`（与 `schema.py` 预留列 + `atomizer.py` 同包），**不**建 `plugins/realityos_quark/` / `plugins/realityos_theory/` 空 plugin 目录——后者会被 plugin loader 探测、可能误激活，且为未实现的 Phase 2 造空壳本身就是一种假绿。单文件 = 纯库模块，无 manifest，不被 loader 当插件，零副作用。Phase 2 落地时建 `realityos_quark`/`realityos_theory` 插件实现这些 Protocol。

## 不做（诚实）

- **Phase 2 全部实现**（QuarkExtractor/TheoryEngine 具体算法、Personal RAG、commitments、G/K/V 三域、ServiceOrchestrator）——母纲「不做」清单，路线图 Phase 2。本 ADR **只钉接口**，兑现 Phase 0 的「定死」承诺，让 Phase 2 不必改 PTG Schema。
- 不接调度、不写 `insight_aggregation`、不消费 `quark_evidence`——全是 Phase 2 行为。

## 后果

- ADR-V6-022 桶 C item C4（Quark/Theory IDL 漏做）**收口**：契约从「只留列无定义」升级为「形状 + 枚举 + 映射 + 版本全定死」。
- Phase 2 启动时，`realityos_quark`/`realityos_theory` 插件实现 `QuarkExtractor`/`TheoryEngine` Protocol 即可，PTG Schema 零迁移（`quark_evidence` 列 + `insight_aggregation` 表早建好）。
- 反假绿：契约明确标注「Phase 2 未实现」，测试守门禁止具体类偷渡进契约模块。
