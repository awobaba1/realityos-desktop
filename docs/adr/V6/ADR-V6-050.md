# ADR-V6-050 — Theory 推导层 + 契约 v2 诚实降级（Phase 2-B B2）

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-039](ADR-V6-039.md) Batch2 + 规则 4（契约变更须 bump）/
[ADR-V6-040](ADR-V6-040.md) D4 诚实降级铁律 / [ADR-V6-032](ADR-V6-032.md) Phase 2 IDL /
[ADR-V6-049](ADR-V6-049.md) B1 quark / 架构 §4.3E line 389 / §4.7 单向数据流

## 背景（theory 实现 + 降级必须机器可读）

ADR-V6-032 钉死了 `TheoryEngine` Protocol + `TheoryDerivation` pydantic（v1），但实现空。
ADR-V6-040 D4 定下 Phase 2-B B2 = `theory LLM 近似（诚实降级）`，且立**铁律**：「theory
输出必须标每 PC 维的 `basis` + `degraded`。**绝不**把 None 渲染成『平稳』或省略降级标」。

问题：v1 `TheoryDerivation` **没有** `basis`/`degraded` 字段。若把降级信息塞进 `rationale`
散文，消费端（UI/CLI）可以**静默忽略**散文，把一个无数据支撑的维度渲染成「平稳」——这正是
铁律警告的最高危假绿（Agent ④ 实证：移植 social_state schema 但不补音频链路 = 表面跑通
Social 维永远空）。降级必须是**机器可读**字段，不是散文。

## 决策

### D1：契约 v2 bump（`PHASE2_CONTRACT_VERSION` 1→2）

`TheoryDerivation` 增两个**可选带默认值**字段（ADR-V6-039 规则 4 合规）：

- `basis: str = ""` —— 该推导的数据依据（哪些原子喂的，或为什么推不出）。
- `degraded: bool = False` —— True = 该推导依赖缺失/严重降级的数据源。

v1 消费者读 v2 记录不受影响（新字段可选）；version bump 是 C1 显式信号。无既有实现，故零破坏。

### D2：`realityos_theory` 插件 —— `TheoryEngineImpl`

满足 `TheoryEngine` Protocol（runtime_checkable 过）。`derive(user_id, atoms, relations)` 经
LLM 近似推导 7 PC + 5 FR 骨架分数，**然后确定性盖章降级**。

### D3：确定性降级图（反假绿核心）

降级由 **engine 决定，不是 LLM**——LLM 不知道自己缺数据。`_PC_DEGRADATION` 确定性映射（基于
ADR-V6-040 line 37 / Agent ④ 实证：无声学只 4/7 PC 可文本推导）：

| PC 维 | degraded | score | basis |
|---|---|---|---|
| Time | False | LLM | 事件时间戳活动节奏文本近似 |
| Emotion | False | LLM | R1/R9 情绪原子 |
| Execution | False | LLM | R2/R12 任务完成率 |
| Cognition | True | LLM（保留） | V6 R8 无连续 cognition score，严重降级，conf=0.25 |
| Energy | True | **强制 0.0** | 需 R1 fatigue+R10 sleep 连续值（Phase 2.5），文本无据 |
| Social | True | **强制 0.0** | 需 Network quark（多人，Phase 2.5+），文本无据 |
| Environment | True | **强制 0.0** | 需 Context quark（声学场景，Phase 3），文本无据 |

**Energy/Social/Environment 强制 score=0.0**：LLM 给的中性 0.5 猜测被丢弃——一个无数据
支撑的维度**绝不能**读出「有测量值」。消费端见 `degraded=True + score=0.0` = 「未测量」，
铁律「绝不把 None 渲染成平稳」由机器字段保证。

FR 5 维：全部 LLM 文本近似（设计如此，非「降级」），`degraded=False` + basis 标注「Phase 2.5
统计公式待替」，conf=0.3。

### D4：单向数据流（架构 §4.7）

`derive` **只读** atoms/relations，写**仅** `insight_aggregation`（PC→`constraint_state`，
FR→`fr_snapshot`，ADR-V6-039 规则 3），**绝不回写**原子层。每条 derivation 独立成行
（period_key=`date|name`），result_data 携带完整 `{score, rationale, basis, degraded}`，
消费端据 `degraded` 诚实渲染。

### D5：C5/C6/C7 全闸（同 quark/insights rails）

`TheoryDerivation` pydantic 门 + `llm_call_logs` 记录 + DLQ 兜底；LLM 失败/JSON 垃圾 → `[]`+DLQ
绝不 raise；prompt `theory_derive_v1.md` 版本化。

## 不做（诚实边界）

- **不**做 Phase 2.5 统计公式（`PC-Energy=σ(R1.fatigue,R10.sleep_deviation)` 等）——需 R10 节律
  连续值表，未建（ADR-V6-039 明示 Phase 2.5）。Phase 2 只 LLM 近似。
- **不**做 Energy/Social/Environment 的 LLM 猜测采信——强制 0.0+degraded，绝不假装测量。
- **不**自动触发（B3 接线调度/CLI；本批 register no-op）。
- **不**回写原子层（单向流铁律）。

## 验证

- `tests/plugins/test_adr_v6_050_theory.py`（13）：契约 v2 version+字段、Protocol runtime_checkable、
  derive 全 12 维、Energy/Social/Environment degraded+score=0、Cognition degraded 保留 score+conf=0.25、
  Time/Emotion/Execution 非降级、FR fr_snapshot 非降级+近似 basis、invalid JSON→[]+DLQ、LLM 异常→[]+DLQ、
  llm_call_log 记录、persist 落 insight_aggregation（7 constraint_state + 5 fr_snapshot）、
  degraded 行 result_data 带 flag + score=0 + conf=0.25。
- ruff 净；CI 三连随提交核验。

## 反假绿（本 ADR 自身的诚实性）

| 项 | 判定 | 依据 |
|---|---|---|
| TheoryEngine 是否真实现 | ✅ | 具体类 + Protocol 过；smoke 实测 derive 12 维 |
| 降级是否机器可读 | ✅ | contract v2 显式 `degraded`+`basis` 字段，非散文 |
| 无据维度是否假装测量 | ❌ 不假装 | Energy/Social/Environment 实测 score=0+degraded |
| 是否在锁内回写原子 | ❌ 不做 | 单向流：只读 atoms，写 insight_aggregation |
| Cognition 是否诚实 | ✅ | degraded=True+保留 score+conf=0.25，标注 R8 无连续值 |
