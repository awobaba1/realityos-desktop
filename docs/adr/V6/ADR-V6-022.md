## 核心结论：四桶分类（不夸不瞒）

### 桶 A — 真做了（核心闭环可信，继续维护）

## 决策（收口总规划）

### 范围裁定（关键前置）

「100% 全量完成」= **Phase 1b 真正收口 + 把桶 B 假绿全修 + 诚实化桶 C + 对外发布可用**。**不含 Phase 2/2.5**（ServiceOrchestrator/Quark/Theory/RAG/声学 SER 按路线图是后续 Phase，ADR-V6-011 已诚实标注「不做」）。依据：用户既定「Phase 1b 广度扩展」方向。

### D1（P0 立即修 — 反假绿，每项一 ADR-V6-023+）

1. **sovereignty 接线到 UI**（ADR-V6-023）：Settings/Memory 加 4 入口（导出 JSON / Mode A 删段 / Mode B 删段 / 未成年人开关）+ 后端 `/api/sovereignty/{export,delete,minor,consent}` 调现成原语。
2. **未成年人模式 Atomizer 集成**（并入 023）：`_materialize_graph` 加 minor gate，`is_minor=True` 跳过 R1SelfState/R9Emotion；`realityos_insights` 聚合同样门控。
3. **post_tool_call 默认 ON**（ADR-V6-024）：V6 bootstrap seed `plugins.enabled` 含 `observability/ptg_capture`，或 migrate_config 加 grandfather 例外。
4. **net-policy 接线**（ADR-V6-025）：`fetch_guard` 挂 web_tools/vision_tools/browser + `assert_llm_egress` 挂 LLM provider base_url。
5. **honcho dormant guard**（并入 025）：默认禁用 + 显式 env 开启 + §7.1 同步叙事（物理删风险高，dormant guard 更稳）。
6. **精度豁免诚实化**（ADR-V6-026）：gate 横幅改「RECALL GREEN / PRECISION DEFERRED」；不新加 precision gate（⑦已证死，precision~70% 是结构天花板）。
7. **insight 缓存键加 prompt_version**（并入 026）：`insight_aggregation` UNIQUE INDEX 加 `schema_version` 入 conflict target + `model_version` 列。
8. **live DB schema 迁移**（ADR-V6-027）：跑迁移修 quality_metrics C2 违规 + 建 ptg_meta；run_eval `--ptg-db` 写入运营库。
9. **time_trend_daily 决策**（并入 027）：**诚实降级**——Phase 1b 不建聚合表（R10/R11 是 Phase 2 特性），改把周镜面 prompt 里涉节律的内容标 deferred，避免无源编造；表本身 Phase 2 起建。

### D2（P1 — Phase 1b 收口）

10. **创始人每日校准 §11.4**（ADR-V6-028）：CLI `realityos calibrate --today`（拉当天原子随机 N 条 → 三分类准/不准/惊喜 → 不准写 feedback 降 confidence + 收集 few-shot → 惊喜进案例库）。这是 1b→2 Gate「纠错率」KR 的可观测性地基。
11. **6 通道验收 ADR 化 + 飞书端到端冒烟**（ADR-V6-029）：钉死验收口径（推荐：**飞书 1 通道端到端冒烟** = 「可用」最低线）+ 桌面加「连接飞书」表单。
12. **run_eval 接 CI**（ADR-V6-030）：nightly 或 release-triggered workflow 带 `DEEPSEEK_API_KEY` secret 跑 `run_eval --limit 30`，精度跌破门禁红。
13. **docker lane 修旁路**（并入 030）：fork 加镜像构建 smoke（不 push），或 fork 专用 workflow；接回 all-checks-pass。
14. **AcousticProvider 诚实降级**（ADR-V6-031）：删架构 §8 line 343-349 Phase 1b 富字段承诺，移 Phase 2.5；`asr_quality_factor` 接口保留（Phase 2.5 填）。**不做** B 路径（现在造 AcousticProvider）——超 Phase 1b 范围。
15. **STT 首启模型预下载**（并入 031）：installer 预拉 faster-whisper base 或 settings 加下载进度。

### D3（P2 — 诚实化/文档/长线）

16. Quark/Theory IDL stub（Phase 2 前，ADR-V6-032）。
17. §7.3 命名契约文档对齐（加「实际落点」列）。
18. §2.1「6 messenger 内置生产级」叙事修正（3 built-in + 3 plugin，非 6 built-in）。
19. dingtalk/yuanbao 出站语音。
20. 样本扩充（真实用户分布）+ match_atom 等价组偏松治理。
21. 桌面默认页加「数据资产首页」选项（把 Memory/Insights 升一等公民）。

### 不做（明确排除，诚实标注）

- Phase 2 全部（ServiceOrchestrator / QuarkExtractor / TheoryEngine / Personal RAG / commitments / G-K-V 三域）—— 路线图 Phase 2。
- Phase 2.5 全部（AcousticProvider SER / 分离 / 韵律 / 声纹 / 生物标记）—— 路线图 Phase 2.5，ADR-V6-011「数月工程」。
- post_gateway_send 出站 hook（D4 Phase 1 不建，sync_turn 已覆盖）。
- §4.10 主动追问（Phase 4）。
- 桶 D 生产验证（需外部用户/真实环境）—— 上线后由真实使用填补，代码侧先备好 telemetry 入口（P1 #12 run_eval CI + #10 校准 CLI）。
