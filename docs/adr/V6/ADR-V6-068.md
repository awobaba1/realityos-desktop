# ADR-V6-068 — 测试真绿修复（pass-only 空桩 + 死模块 skip）

**状态**: Accepted
**日期**: 2026-07-21
**关联**: [ADR-V6-053](ADR-V6-053.md) 第三轮审计 R6 / 铁律 C4 / [ADR-V6-006](ADR-V6-006.md)（aux-vision 禁用）

## 背景（第三轮审计 R6：两个零信号测试）

第三轮审计 R6 发现两处「测试不测试」——零运行时信号，是假绿的测试层变体：

### 问题 1：`test_marker_message_inserted_when_missing` 是 pass-only 空桩

`tests/run_agent/test_tool_call_args_sanitizer.py:87-91`：
```python
def test_marker_message_inserted_when_missing():
    # Removed May 2026 — pre-existing assertion mismatch on origin/main
    # (the dict ordering or marker shape changed without test update).
    # Deleted wholesale per Teknium's keep-CI-green instruction.
    pass
```
**零断言、永远绿**——Teknium 上游「keep-CI-green」遗物。真实测试被 `_disabled_` 前缀禁用（line 94-110）。

**根因查实**：禁用测试用精确字典 `==` 断言插入的 tool 消息：
```python
assert messages[1] == {"role": "tool", "name": "read_file",
                       "tool_call_id": "call_1", "content": marker}
```
但 `make_tool_result_message`（agent/tool_dispatch_helpers.py:457）现在多吐 `tool_name` 键（可能还有 `_tool_output_risk`），精确 `==` 因此挂掉。注释「dict ordering or marker shape changed」即此——不是 ordering，是**附加键**。

### 问题 2：`test_computer_use_capture_routing.py` 整模块 `pytestmark = skip`

468 行测试整模块跳过（ADR-V6-006 skip）。skip 注释**写得诚实**（声称覆盖移到 `test_computer_use_v6_boundary.py`），且经核实替代覆盖**真实存在**（boundary 文件 19 个 test，覆盖当前禁用边界：`_should_route_through_aux_vision() is False` + 云视觉白名单门 + `_multimodal` 处理）。

但 468 行永不执行的测试 = **假覆盖信号**：读测试套件的人看到 468 行「测试」却零运行时信号。且 aux-vision 路由 **永久出 RealityOS 范围**（CLAUDE.md 铁律「除 GPS 外不多模态 / 无相机」）——该「未来重启」永不发生。

## 决策

### D1：sanitizer pass-only → 真绿子集断言

把 pass 空桩还原为**真实断言**，用**子集检查**（钉契约：插入的合成 tool 消息 role/name/tool_call_id/content + 后续 user turn 保留）替代脆的精确 `==`：
```python
assert messages[1]["role"] == "tool"
assert messages[1]["name"] == "read_file"
assert messages[1]["tool_call_id"] == "call_1"
assert messages[1]["content"] == marker
assert messages[2] == {"role": "user", "content": "next turn"}
```
子集检查对 `tool_name`/`_tool_output_risk` 附加键不脆。删 `_disabled_` 重复函数。

**内容不经包装核实**：`read_file` 不在高风险表（web_extract/web_search/browser_*/mcp_*），marker 是短串 → `_maybe_wrap_untrusted` 原样返回 → `content == marker` 成立。

### D2：删除 computer_use 死模块（非保留诚实 skip）

删除 `tests/tools/test_computer_use_capture_routing.py`（468 行）。理由：
1. **替代覆盖已证实**：`test_computer_use_v6_boundary.py` 19 测试覆盖当前禁用契约。
2. **零信号**：skipped 测试无法检测回归——保留即假覆盖信号。
3. **永久出范围**：aux-vision 受 CLAUDE.md 铁律排除（除 GPS 外不多模态），「重启」永不发生；为永不到来的未来保留 468 行死代码正是假绿温床。

不保留「诚实 skip」——一个永不执行的 468 行文件无论注释多诚实，仍是死重 + 误导。删它是诚实地板。

## 不做（诚实边界）

- **不**改 `make_tool_result_message` 加 `tool_name` 的设计（那是 wire 格式 + session DB 双写需要，正确）。测试适配实现，非反之。
- **不**为 aux-vision 写任何新测试——该功能永久出范围，写测试才是假绿（测一个永不启用的功能）。
- **不**碰 `test_computer_use_v6_boundary.py`——它已是真覆盖，无需动。

## 反假绿
