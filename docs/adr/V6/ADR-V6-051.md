# ADR-V6-051 — Phase 2-B B3 接线：quark/theory CLI + theory 启动惰性调度

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-040](ADR-V6-040.md) 统一规划（Phase 2-B B3）/
[ADR-V6-049](ADR-V6-049.md) B1 quark 闭循环 / [ADR-V6-050](ADR-V6-050.md) B2 theory 闭循环 /
[ADR-V6-019](ADR-V6-019.md) insights 启动惰性调度（镜像源）/ 架构 §4.7 单向数据流 /
铁律：分层（下层不调上层）、C7 无静默失败、C2 不可丢失

## 背景（B1/B2 闭循环存在但只测试可达）

ADR-V6-049（quark）与 ADR-V6-050（theory）交付了**闭循环**实现 `extract_and_aggregate` /
`derive_and_persist`，但两者 `register` 均 no-op，明确标注「自动触发是 B3」。即 Phase 2-B 的
**调度接线 + CLI 入口**尚未落地——用户/桌面大脑无法实际触发 quark 抽取或 theory 推导，只能跑测试。
这正是 ADR-V6-037 的「做了没发 = 最致命假绿」模式在生产外的变体：**实现了却不可达 = 假绿**。
ADR-V6-040 把 B3 列为 Phase 2-B 的收口项；本 ADR 兑现它。

## 决策

### D1：两个 CLI 子命令（`hermes quark extract` / `hermes theory derive`）

按 ADR-V6-047/048（memo/people）既定接线模式落地两个子命令：

- `hermes quark extract <memo_id>` —— 加载 memo（新增 `PTGStore.get_memo`），取 `effective_text`
  （`corrected_text ?? source_text`，与 A4 公式同源），调 `extract_and_aggregate`（空 evidence
  rows——Phase 2 文本级，不伪造声学/多人证据），打印「抽出 N / 聚合 M」。
- `hermes theory derive [--period-key]` —— 采集 `recent_atoms` + `relations_for_user`，调
  `derive_and_persist`，打印「推导 N / 落库 M / 降级 K」。

I/O 与业务逻辑分离：CLI handler 仅 open store / resolve founder / 调闭循环 / 打印；闭循环逻辑在
plugin，单元测试免 tty、免真实 LLM（monkeypatch 闭循环入口）。退出码语义：**命令跑通即 0**（0
quark / 0 derived 是诚实空结果，非失败）；仅 memo 找不到等操作性失败返 1。

### D2：`PTGStore.get_memo(user_id, memo_id)`（D1 的前置缺口）

仓库原先**无**按 id 取单 memo 的方法（仅 `correct_memo_source_text` 内联 SELECT，且只返状态字段）。
新增 `get_memo`：返回 `{id, source_text, corrected_text, version, effective_text}`；**软删行返 None**
（C2：deleted 对读者不可见）；C7 读不 raise。

### D3：theory 启动惰性调度（`plugins/realityos_theory/scheduling.py`）

**镜像** insights 的启动惰性调度（ADR-V6-019）——`realityos_theory` 自有自己的 scheduling 模块
（Option A，非塞进 insights 的 `_SPECS`），因 insights 调 theory 会破分层铁律；theory 自己调度自己。

- **触发**：`register()` → `start_scheduler_if_due` → 守护线程；进程级 once-guard + daemon。
- **门控 `run_due_theory`**：以规范探针行 `(user, constraint_state, "{period}|Time")` 探缓存——
  存在且 `schema_version == PROMPT_VERSION` 则**跳过**（幂等，不重烧 LLM）；旧 prompt 版则**重生成**
  （stale-refresh，ADR-V6-026）；空原子图则**冷启动跳过**（不空烧 LLM）。
- **周期粒度**：当日北京时间日期（`%Y-%m-%d`）；`beijing_now` **内联**（不 import insights），消除
  theory→insights 横向依赖。
- **opt-out**：`PYTEST_CURRENT_TEST` 或 `REALITYOS_THEORY_AUTOSCHED=0|false|no|off` → 不启动（测试/瞬态
  CLI 永不空烧真实 LLM，反假绿）。
- **fail-open**：全程 C7，调度器永不 raise；`derive_and_persist` 抛错 → `{reason:"error"}` 不逃逸。

### D4：单向数据流（架构 §4.7）+ 分层铁律

- theory 调度**只读** atoms/relations，写**仅** `insight_aggregation`，绝不回写原子层（与 B2 同源）。
- theory 调度依赖**仅** memory 层（PTGStore）；`register` 是 plugin 自身的注册点，下层 memory 永不
  反向调用 theory。分层铁律由测试 `test_no_insights_dependency_at_import`（AST 扫 import 语句）锁死。

## 不做（诚实边界）

- **不**做 cron / 外部调度器——桌面大脑不一定在整点开机，启动惰性「开机即补」够用（同 insights）。
- **不**做 quark 的自动调度——quark 是**捕获时**抽取（随 memo/工具事件实时），无周期性可补；只有
  theory 是周期性「重推导」（PC/FR 随原子图演化而刷新），故仅 theory 有调度。
- **不**做 quark evidence rows 的 CLI 注入——`<memo_id>` 级证据无 memo→tool_events 桥（确认无
  `tool_events_for_memo`），Phase 2 文本级抽取消费用空 rows；声学/多人证据是 Phase 2.5+。
- **不**改 B2 的 `derive_and_persist` 内部（仅接线，不返工）；hardcoded `schema_version="v1"` 与
  `PROMPT_VERSION="v1"` 当前一致，bump 时统一处理。

## 验证

- `tests/plugins/test_adr_v6_051_b3.py`（14）：
  - `get_memo`：happy/not_found/软删返 None/corrected 优先；
  - `run_due_theory`：冷启动跳过 LLM / 缺失则生成 / 当前 prompt 存在则跳 / 旧 prompt 重生成 / derive 抛错隔离；
  - once-guard / disabled 不启动 / env opt-out / pytest 禁用；
  - 分层 AST：scheduling 不 import insights。
- `tests/hermes_cli/test_quark_cmd.py`（5）：happy/not_found/空文本/0 quark 非失败/usage。
- `tests/hermes_cli/test_theory_cmd.py`（4）：happy/无原子/0 derived 是 DLQ/usage。
- ruff 全量净；`hermes quark/theory --help` 实渲染；`register()`（AUTOSCHED off）不 spawn；CI 三连随提交核验。

## 反假绿（本 ADR 自身的诚实性）

| 项 | 判定 | 依据 |
|---|---|---|
| B1/B2 闭循环是否真可达 | ✅ | 两个 CLI 子命令 + 调度触发，`--help` 实渲染、handler smoke 过 |
| get_memo 是否真实现 | ✅ | 具体 SQL + 软删 None + 4 测试覆盖 |
| 调度是否真接 register | ✅ | register() 实调 start_scheduler_if_due；门控探针实测 4 路径 |
| 是否塞进 insights 破分层 | ❌ 不做 | Option A 自有模块；AST 测试锁死无 insights import |
| 调度是否空烧 LLM | ❌ 不空烧 | 冷启动跳过 + 探针幂等 + pytest/env opt-out |
| 0 结果是否当成功掩盖 | ❌ 不掩盖 | 0 quark/0 derived 打「未抽出/未产出（DLQ）」诚实文案 |
| 单向流是否守住 | ✅ | theory 只读 atoms，写 insight_aggregation；调度测同源 |
