# ADR-V6-030：run_eval 进 CI（召回回归门禁）+ docker lane 接回 all-checks-pass

- **状态**：Accepted（Phase 1b 收口·批次 2，2026-07-20）
- **日期**：2026-07-20
- **Phase**：1b
- **关联**：ADR-V6-022（母纲，桶 B-B9/B10、决策 D2#12/#13）、ADR-V6-026（精度豁免诚实化——本 ADR 继承其「门禁=recall only」）、ADR-V6-029（同批）
- **铁律**：C1（本决策一 ADR）、C3（生产现实即真理——CI 须能看见召回回归）、C7（无静默失败——门禁红须 exit 非 0；桶D 休眠须显式 notice 非静默 skip）

---

## 背景

ADR-V6-022 审计暴露两条 CI 假绿（桶 B）：

- **B9**：CI 关键路径全 mock + `run_eval` 不进 CI → **精度/召回回归 CI 永远发现不了**。
- **B10**：docker lane 剔除 `all-checks-pass` + fork 永远 skip → **镜像从不在 CI 验证**。

母纲决策（line 99-100）：D2#12「run_eval 接 CI（ADR-V6-030）：nightly 或 release-triggered workflow 带 `DEEPSEEK_API_KEY` secret 跑 `run_eval --limit 30`，门禁跌穿红」；D2#13「docker lane 修旁路（并入 030）：fork 加镜像构建 smoke（不 push），或 fork 专用 workflow；接回 all-checks-pass」。

本 ADR 分两 commit 落地：Commit A 修 B9（低风险、可单测），Commit B 修 B10（较险、CI 首跑即真相）。

## 关键口径纠正（反假绿）

母纲 D2#12 措辞「精度跌穿门禁红」**技术上不准**。ADR-V6-026 已落地（commit f0f4ffa69）的事实是：

- **门禁 = RECALL ONLY**。recall per-type 跌穿阈值（R3 0.85 / R2 0.80 / R1 0.70 / R7·R8·R9·R12 0.60）或 `errors>0` → 红。
- **precision 永不过门**。⑦ ~70% 是结构天花板（commit 7758542e0 证死），非回归。横幅永远 `RECALL GREEN / PRECISION DEFERRED`，绝不 `ALL GREEN`。

故本 ADR 的 CI 门禁**只能因召回回归红**，precision 数字再低也不红。这是 ADR-V6-026 决策 1「不新加 precision gate」的延续。本 ADR 不违背、只继承。

## 决策

### D1：run_eval 加 `--ci-gate`（exit-code 接缝）— Commit A

`tests/benchmark/run_eval.py` 此前跑完 `run_evaluation()` 即返回，**门禁红也不 exit 非 0** → CI 无法据其红。修：

- 新增纯函数 `ci_gate_exit_code(report) -> int`：`report["gate_all_green"]` 为真返回 0，否则 1。precision 字段**不参与**（见上口径纠正）。
- 新增 `--ci-gate` flag：`__main__` 捕获 `report = run_evaluation(...)`，flag 置真时 `sys.exit(ci_gate_exit_code(report))`。
- 回归测试 `tests/benchmark/test_run_eval_ci_gate.py`：纯函数测试，**无 LLM 无网络**，钉死契约——green→0 / red→1 / 低 precision 不红 / 空报告防伪装。

抽纯函数是为可测（CI lane 本身要真 DeepSeek 无法本地/CI mock 跑；纯函数把契约从 LLM 依赖中剥离，独立回归）。

### D2：nightly eval lane — Commit A

新增 `.github/workflows/eval-nightly.yml`：

- 触发：`schedule: '17 16 * * *'`（16:17 UTC ≈ 00:17 北京，off-minute 避 fleet :00 拥堵）+ `workflow_dispatch`（手动，可调 `--limit`）。**非 PR 门禁**（run_eval 烧真 LLM 调用，做观测面非每 PR 检查）。
- Setup：mirror CI `tests.yml`（checkout v6.0.2 / setup-uv 8.2.0 / python 3.11 / `uv sync --locked --extra all --extra dev`）。
- 跑：`python -m tests.benchmark.run_eval --limit 30 --workers 8 --ci-gate --out eval-report.json`，`DEEPSEEK_API_KEY`/`DEEPSEEK_BASE_URL` 从 secret 注入。
- 上传 `eval-report.json` artifact（retention 30 天）。

**桶D 边界（反假绿核心）**：lane 默认 **DORMANT**——无 `DEEPSEEK_API_KEY` secret 时打印 `::notice::` 说明「桶D，加 secret 激活」并 `exit 0`。这是**诚实休眠**（exit 0 + 显式 notice 记录 skip 原因），**非假绿**（假绿是声称 eval 跑过且过；本 lane 明确说没跑因无 key）。一旦用户加 secret，lane 即真跑 recall 门禁，红则 exit 1。

### D3：docker-build-smoke + 接回 all-checks-pass — Commit B

B10 根因（亲核确认）：
- `docker.yml:25` `build` job `if: github.repository == 'NousResearch/hermes-agent'` + `:158` `merge` job 同守卫 → fork（`awobaba1/realityos-desktop`）两 job **永远 skip**。
- `ci.yml:160-161` `all-checks-pass` 的 `needs:` 里 `- docker` 被注释（「we don't require docker to pass rn because it's so slow lol」）→ 即便不 skip 也不进必过门。

修（Commit B）：
- **新增 `docker-build-smoke` job**（放 ci.yml，非改 docker.yml matrix）：amd64 only、`docker build`（no push、no 全量集成测试）、**无 fork-skip 守卫**（fork 每 PR 跑）。复用 `docker/setup-buildx-action@v3` + gha cache。这是 B10 核心「镜像从不在 CI 验证」的对症——现在每 PR 验证 Dockerfile 可构建。
- **ci.yml `all-checks-pass` 加 `- docker-build-smoke`** 进必过门。
- **不动 `docker.yml`** 的 `build`/`merge` matrix（那是上游 push+Docker Hub 凭据路径，fork 无凭据；smoke 是独立轻量验证）。

**风险与诚实处置**：本地 docker daemon 未运行，无法预验 Dockerfile 构建。故 Commit B 的 CI 首跑即**首次真实验证**——绿则收口，红则 CI 已诚实暴露 Dockerfile 问题，前向修复（非假绿，CI 即真相）。

## 验证

- **D1 纯函数**：`tests/benchmark/test_run_eval_ci_gate.py` 本地绿（5 测，无 LLM）。
- **D2 lane 结构**：workflow YAML 语法 + setup mirror CI；dormant 分支无 secret 时 exit 0 + notice（手动 dispatch 可观测）。
- **D3 smoke**：Commit B 的 CI run 即验证（build 绿 or 诚实红）。
- **三连核验**：每 commit headSha == CI `gh run view --json conclusion` 绿 + 内容指纹。

## 后果

- B9 闭：召回回归从「CI 永远看不见」变「nightly 真跑 + `--ci-gate` exit 1」。precision 仍不过门（ADR-V6-026 继承），但作为数字报告在 banner + artifact，可观测。
- B10 闭：镜像每 PR 至少构建验证一次，且进必过门（Commit B 后）。
- 桶D 诚实：eval lane dormant 状态显式 notice；激活须用户加 `DEEPSEEK_API_KEY` Actions secret。

## 延迟项（诚实）

- **激活 eval lane**：用户须在 `awobaba1/realityos-desktop` 加 `DEEPSEEK_API_KEY`（可选 `DEEPSEEK_BASE_URL`）Actions secret。未加前 lane dormant（exit 0 + notice），不假绿也不报红。
- **docker-build-smoke 进必过门的稳定性观测**：Commit B 首跑后须观察若干次稳定绿；若 Dockerfile 在 fork 有构建问题，前向修（CI 即真相）。
- **release-triggered eval**：现仅 schedule+dispatch；若要 release 前全量 200 样本跑，加 `on: release: types: [published]`（按需）。
