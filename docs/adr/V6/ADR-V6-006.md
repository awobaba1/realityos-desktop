# ADR-V6-006: Phase 0 Strip 终版——单点 stub + DORMANT + 电脑操作 6 口子必堵

- **Status**: Accepted
- **Date**: 2026-07-17
- **Refines**: ADR-V6-005（strip 范围，REV-9 已纠正其 2 处事实错误）
- **Refs**: C2 不可丢失 / C7 无静默失败 / ADR-V6-002 操作能力继承 / ADR-V6-003 数据边界 / REV-9 H2 / H9 / 决策 D2

## Context
ADR-V6-005 把 strip 从「破坏性删除」降级为「惰性化 + 继承」。REV-9 全量精读（6 路 agent 逐行）进一步确认：
- conversation_loop 有 **77 处** Nous 引用（全 `provider==nous` 门控不执行），但单一 kill switch 即可让 25+ 调用方 fail-closed，**无需删 77 处**。
- 7 个云 memory provider 代码本体保留无害（不配置即不加载）。
- 电脑操作（computer_use）默认配置有 **6 个 P0 数据边界口子**，违 ADR-V6-003「流经即捕获但不偷窥屏 + 图像不沉淀」，**Phase 0 必堵**。

本 ADR 把 strip 终版化，并新增电脑操作边界堵口。

## Decision

### 1. Nous — 单点 stub（不删 77 处）
- `hermes_cli/nous_account.py:322` `get_nous_portal_account_info` 返回 `logged_in=False`（stub）。
- 25+ 调用方全部 fail-closed；77 处门控分支永不进入（V6 用智谱 GLM-5.2/DeepSeek，`provider != "nous"`）。
- **不删** conversation_loop 的 Nous 代码（删了风险大、收益零）。

### 2. 云 memory — DORMANT（不配置即不加载）
- V6 `config.yaml` **不带** `memory.provider` 指向任何云 provider。
- 7 云 provider（byterover / hindsight / holographic / honcho / mem0 / openviking / retaindb / supermemory）代码保留，零加载。
- PTGProvider 是唯一 active provider（`agent/memory_manager.py:add_provider` 强制 max-1-external）。

### 3. KEEP（身体功能，ADR-V6-002）
- `trajectory_compressor`（上下文/轨迹压缩）
- `batch_runner`（eval harness，V6 benchmark 复用）
- `background_review`（skill-write 审批后台，local 自评非外联）
- honcho 插件代码本体（DORMANT，**不铲除**——避 10 处引用断链；取代 §6.4 旧叙事「exclusive 铲除」）

### 4. 电脑操作 6 个 P0 边界口子必堵（H9，REV-9）
| # | 口子 | 位置 | 堵法 |
|---|---|---|---|
| 1 | `capture` 截图列为免审动作 | `tools/computer_use/tool.py:80` `_SAFE_ACTIONS` | 从 _SAFE_ACTIONS 移除 capture，截图必走审批 |
| 2 | 无 callback 默认 allow | `tools/computer_use/tool.py:311-315` | 改默认 **deny**（无显式审批则拒绝） |
| 3 | 多模态信封上送截图 PNG 到云端 LLM | `tools/computer_use/tool.py:657-667` | 主模型本地化 / 显式云模型白名单 + 截图脱敏 |
| 4 | aux-vision 送外部 Gemini | aux-vision 路径 | 关闭 aux-vision（内测期不依赖外部视觉模型） |
| 5 | `start_recording` 录整屏 mp4 | `tools/computer_use/cua_backend.py:2173` | 不暴露给 agent（从工具 schema 移除） |
| 6 | SKILL.md 教写截图到磁盘 | `skills/computer-use/SKILL.md` | 删「delivering screenshots / 写磁盘」指引 |

### 5. computer-use 内测期默认关（D2）
- 工具默认 **disabled**，设置里显式开启。
- 开启后每次执行仍走 `tools/approval.py` 审批门（逐次授权）。
- 读屏图像**不沉淀** PTG（ADR-V6-003）。

## Consequences
- ✅ Nous 零断链（单点 stub vs 删 77 处）。
- ✅ 云 memory 零加载，主权层干净。
- ✅ 电脑操作数据边界堵死，截图不出境、不沉淀。
- ✅ 身体功能全保留（fork-inherit 哲学）。
- ⚠️ 仓库体积含惰性代码（可接受，v2 清理）。
- ⚠️ computer-use 默认关使内测期能力展示弱（D2 取舍：稳 > 炫）。
