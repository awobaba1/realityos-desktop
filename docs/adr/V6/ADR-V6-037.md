# ADR-V6-037：desktop-build 自动挂载 Release 资产——消除「做了没发」manual gap

- **状态**：Accepted（Phase 1b 收口·批次 3 后，2026-07-20）
- **日期**：2026-07-20
- **Phase**：1b（分发/反假绿）
- **关联**：ADR-V6-010（分发模型）、ADR-V6-022（母纲，桶 D item D3「真实端到端呈现」的可分发前置）、ADR-V6-015（CI 红收口/真绿 Release）
- **铁律**：C7（无静默失败——收口工作静默滞留 main 未发布=最致命的静默「成功」）、C3（生产现实即真理——main 绿≠用户可下载）

---

## 背景

批次 3 收口后核验发现：**批次 1/2/3 共 26 个提交（ADR-027~036 全部反假绿收口）在 main 且 CI 全绿，但 NOT 进任何 Release**。最新 Release v2026.7.20 的 tag 指向 `0bcbe46ff`（批次 3 之前），用户实际能下载的版本不含：

- ADR-027 quality_metrics 治愈 + time_trend 诚实降级
- ADR-028 创始人每日校准 CLI
- ADR-029 6 通道 L1 验收 / ADR-030 run_eval ci-gate / ADR-031 AcousticProvider 降级
- ADR-032 Quark/Theory IDL / ADR-033 spec 诚实化 / ADR-034 语音矩阵 / ADR-035 match_atom 治理 / ADR-036 桌面默认页

**这是最致命的假绿**：报告「批次 3 收口完成」属实（main 上真做了、CI 真绿），但**分发物里没有**——用户拿到的仍是旧版本。根因：`desktop-build.yml` 只产 artifact（30 天过期），**artifact→Release 挂载是 manual step**（历史 Release 靠人工 `gh run download` + `gh release upload`），批次 3 后没人执行这一步。

## 决策

### D1：desktop-build.yml 加 `publish` job（CI 内自动 artifact→Release）

新增 `publish` job：
- `needs: build`（两个 OS 矩阵都绿后才发）。
- `if: startsWith(github.ref, 'refs/tags/v')`——**仅 tag 触发**（workflow_dispatch 测试构建无 Release 可挂，仍只产 artifact）。
- `permissions: contents: write`（job 级覆盖 workflow 级 `contents: read`，`gh release upload` 需要）。
- 步骤：`actions/download-artifact@v4`（`merge-multiple: true` 拍平 mac-dmg/+win-nsis/）→ `gh release create`（若不存在，幂等）→ `gh release upload --clobber`（dmg+exe）→ 打印资产清单。

### D2：发布流程从「tag→build→人工挂载」变为「tag→build→自动挂载」

1. （可选）workflow 改动先 push main，ci.yml 跑 lint 校验 YAML。
2. `gh release create v<X> --target main --generate-notes` 创建 Release + tag（tag push 触发 desktop-build）。
3. build 矩阵产 mac dmg + win exe artifact。
4. **publish job 自动下载两 artifact + 挂到 Release**（无需人工、无需本机 256MB 往返——CI runner 内完成，避开本机→GitHub blob egress 慢/超时假 HTTP000 陷阱，见 [[proxy-config]]）。
5. 三连核验：Release 资产存在 + size 与本地基线一致 + 公开下载 HTTP206。

### D3：文件名版本（0.17.0）vs Release tag（v2026.7.X）惯例保持

历史 Release（v2026.7.18/19/20）资产文件名都是 `RealityOS-0.17.0-*`（package.json 版本跨 Release 不变），**Release tag（v2026.7.X）才是用户可见版本号**。本 ADR 不改 package.json 版本（保持惯例；版本 bump 是独立决策，非发布管线范畴）。

## 不做（诚实）

- **不在 workflow_dispatch 构建 publish**（无 Release 可挂；测试构建只产 artifact）。
- **不改 package.json 版本**（D3：惯例保持；文件名版本 ≠ Release 版本）。
- **不动既有 Release**（v2026.7.20 不重打 tag——消费者可能已缓存，移动已发布 tag 是反模式；新工作发新 Release v2026.7.21）。
- **不跳过三连核验**（publish job 绿 ≠ 资产可下载；须独立核 hash + size + HTTP206）。

## 后果

- **批次 1/2/3（ADR-027~036）随 v2026.7.21 进分发**——「100% 生产级对外发布可用」名副其实。
- **永久消除 manual gap**：未来 tag → 自动 build + publish，收口工作不再可能「绿但没发」。
- 反假绿：C7 最严形态——「声称完成却不可达」被结构性堵死（发布是 CI 的必经关节，非靠人记）。
- publish job 首次运行即验证（workflow 自身约定「first tagged run is green-signal」）。

## 验证

- desktop-build.yml push main → ci.yml 绿（YAML lint）。
- `gh release create v2026.7.21 --target main` → tag push 触发 desktop-build。
- build 矩阵（macos + windows）绿 + publish job 绿。
- 三连核验：Release v2026.7.21 资产 `RealityOS-0.17.0-mac-arm64.dmg`（~138MB）+ `RealityOS-0.17.0-win-x64.exe`（~118MB）存在 + size 与基线一致 + 公开下载 HTTP206。
