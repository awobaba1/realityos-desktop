# ADR-V6-017：Phase 1b-2 — 每周镜面（Weekly Mirror）

- **状态**：Accepted（Phase 1b 洞察层，2026-07-19）
- **日期**：2026-07-19
- **Phase**：1b（广度优先，首个 INSIGHT 产品）
- **关联**：架构 §0.5③（冷启动门控）/ §4.4（洞察缓存）/ §18.5 #4（prompt 模板）、ADR-V6-011（Phase 1a 心脏，原子层基座）、ADR-V6-016（Phase 1b-1 R8/R9/R12，镜面的原料）、ADR-V6-008（PTGStore 共享单例）
- **铁律**：C1（本 ADR）、C2（复用 insight_aggregation 缓存表，零新表；喂镜面的原子不可变）、C5（镜面 C5 文本门槛）、C6（weekly_mirror_v1 版本化 prompt + 每次 LLM 调用入 llm_call_logs）、C7（全路径 fail-open + DLQ + 占位降级）

---

## 背景

Phase 1b-1（ADR-V6-016）把原子层从 5 类扩到 8 类（R0/R1/R2/R3/R7 + R8/R9/R12）+ ConfidenceEngine Stage2。原子层会"记"，但还不会"照见"。

架构 §0.5③ 把**每周镜面**列为 Phase 1b 头号留存特性，并给出一条生死门控：

> 注册 < 14 天 **或** memo < 15 条 → **不发周报**，发引导占位。否则新用户 Day3 看到"你这周提了 0 次家人"直接卸载。

§18.5 把镜面定为 PRD #4 prompt 模板（Phase 1b）。§4.4 提供了现成的 `insight_aggregation` 缓存表（带 `data_sufficiency` 枚举、TTL、唯一键）——本 ADR 发现它**正是为此而建**，无需新建表。

## 决策

### 1. 新插件 `realityos_insights`，服务为核心

- **`WeeklyMirrorService`**（`plugins/realityos_insights/weekly_mirror.py`）是可注入、可测的生产核心：收一个共享 `PTGStore`（ADR-V6-008 单例）+ `caller`（LLM，可注入 mock）+ `now_fn`（时钟，可注入）。
- `register(ctx)` 是显式调用面（同 `realityos_sovereignty`）——服务由桌面 UI / 调度作业显式调用，**不**在本批假绿接线（接线是诚实记录的下一步）。

### 2. 流程：聚合 → 冷启动门 → LLM → C5 → C6 → 缓存

`generate(user_id, *, week_start=None, generated_by='scheduled')`：

1. **周窗口**：默认 = 北京时间最近一个**完整**的周一~周日。events `timestamp` 存 UTC，故周边界按北京本地算后转 UTC 查询（半开 `[ws, we)`）。
2. **冷启动门（§0.5③）** → `data_sufficiency` + `status`：
   - 注册 < 14 天 **OR** 总 memo < 15 → `insufficient` / `placeholder`（**不发 LLM**）
   - 15 ≤ memo < 30 → `partial` / `mirror`（prompt 提示"结论要软"）
   - memo ≥ 30 → `sufficient` / `mirror`
3. **聚合**（`aggregation.aggregate_week`）：窗口内 8 类原子按类型计数 + 样本（top 人物 / 任务 / 任务结果分布 / 情绪 / 认知 / 自我状态 / 表达 / top 实体）。
4. **非 insufficient**：一次 LLM 调用（`weekly_mirror_v1` prompt + 聚合 JSON，temperature 0.4，markdown 输出非 JSON）→ 镜面正文。
5. **C5 文本门槛**（`_validate_mirror`）：strip 后 ≥ 80 字 + 不以 `{`/`[` 开头（防 JSON 泄漏）+ 非空。不达标 → DLQ + 降级占位。
6. **C6**：每次 LLM 调用入 `llm_call_logs`（`prompt_template_version='v1'`、`schema_valid`、tokens、latency、success）。
7. **C7**：LLM 异常 / C5 失败 / 聚合失败 / 写入失败 —— 全路径 fail-open，记 DLQ，降级为占位，**绝不向调用方抛异常**。
8. **缓存**：`upsert_insight(aggregation_type='weekly_mirror', period_key=<周一日期>, ...)`。

### 3. 复用 insight_aggregation，零新表（C2）

`insight_aggregation` 表（Phase 1a 预建）字段与本场景逐一对齐：

| 表字段 | 镜面用途 |
|--------|---------|
| `aggregation_type='weekly_mirror'` | 类型标记 |
| `period_key` / `period_start` / `period_end` | 周一日期 / UTC 窗口 |
| `input_data` | 聚合 JSON（可复算） |
| `result_data` | 镜面正文（或占位） |
| `data_sufficiency` | **冷启动门控结论**（sufficient/partial/insufficient） |
| `generated_by` | scheduled/manual/on_demand |
| `llm_call_id` | C6 traceability |
| `version` / `expires_at` / `deleted_at` | C2 + TTL |

唯一索引 `(user_id, aggregation_type, period_key)` ⇒ 每用户每周一行；重算同周用 `ON CONFLICT DO UPDATE` **原地替换**（`version+1`）。这是正确的缓存语义——**喂镜面的原子在事件表里不可变（C2 在那保护）**，镜面本身是派生缓存。

新增 PTGStore 方法（皆 fail-open、可测）：
- `recent_atoms(..., since=None, until=None)` —— 时间窗（半开，UTC ISO）扩展到全部 4 个事件表查询。
- `memo_count(user_id)` —— 冷启动门控的 memo 计数（全量、非窗口）。
- `user_created_at(user_id)` —— 注册时间（门控的注册天数）。
- `upsert_insight(...)` —— ON CONFLICT 原地替换。

SCHEMA_VERSION **不变**（6）——纯复用既有表 + 新插件代码。

### 4. prompt #4 版本化（C6）

`plugins/realityos_insights/prompt_templates/weekly_mirror_v1.md` —— 独立 v1，永不覆盖。铁律：具体 > 泛泛、照见不评判、不说"你总是"、`partial` 更软、不捏造、隐私语气。缓存键含 `prompt_version` → 改模板自动失效旧镜面（§4.4④）。

### 5. 降级占位的两种语义

- **冷启动占位**（`insufficient`）：§0.5③ 引导文案"我还在了解你…"。这是**正常**路径，不是错误——发 LLM 才是错的。
- **错误占位**（LLM 失败 / C5 失败）：文案"这周的镜面我还没准备好…"。**始终**记 DLQ + llm_call_logs success=0/schema_valid=0（C7 无静默失败）。

两种占位都写入 insight_aggregation（含 `llm_call_id` / `data_sufficiency`），让 UI 一致展示 + 调度不重复猛打 LLM。

## 验证

- **17 项测试全绿**（`tests/plugins/realityos_insights/test_weekly_mirror.py`）：
  - 聚合：窗口边界（窗外原子排除）+ 8 类型 + 人物分组 + 任务结果分布 + 空周。
  - 三门控分支：insufficient(via memo) / insufficient(via 注册) / partial / sufficient（placeholder 路径**零 LLM 调用**）。
  - LLM 成功：C6 入库（v1 / success / schema_valid=1）。
  - LLM 失败 + C5 失败：降级占位 + DLQ + llm_call_logs success=0。
  - 存储：insight_aggregation 落库 + 重算原地替换（version→2，仍 1 行）。
  - 窗口：默认 = 上一完整周；显式 week_start。
  - 端到端：真实聚合数据（张三/述职/React/愤怒）进 prompt。
- **全量 memory+insights 套无回归**（recent_atoms 时间窗扩展向后兼容：since/until 默认 None = 原行为）。
- **ruff 全量净**。

## 后续（Phase 1b-3+）

1. **调度接线**：每周一 09:00 启动检查（桌面大脑可能没开 → 同 backup 的 startup-lazy 模式，ptg_meta 时间戳判过期）。
2. **桌面 UI 渲染**：读 insight_aggregation 当周行 → 渲染镜面/占位（sovereignty 的 UI 接线同类项）。
3. **每日报告 #2**（§18.5）：同框架，aggregation_type='daily_report'。
4. **真实 LLM 端到端冒烟**（C3）：用真实 DeepSeek 跑一份镜面，人工审读"具体/不说教/不捏造"（本批用 mock 验全流程，真实 LLM 质量是 Phase 1b-3 接线后的事）。
5. **创始人校准回流**（§11.4）：镜面接受"不对"反馈 → 回流到提取质量基准。
