# ADR-V6-064 — post_gateway_send 捕获点 defer 正式化（消除 ADR-V6-003 假 D4 借口）

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-053](ADR-V6-053.md) P2-T5 / [ADR-V6-003](ADR-V6-003.md) / 铁律 C1（一切皆决策 — 不存在的决策借口=假绿） / [ADR-V6-037](ADR-V6-037.md) 反假绿 / [ADR-V6-063](ADR-V6-063.md)（同类 T5 碎片：citation 写了无消费）

## 背景（审计发现 post_gateway_send 假 D4 借口）

ADR-V6-003（ptg_capture 母纲）第 11 行**设计要求** post_gateway_send 作为「agent 主动出站」的捕获点（标注「新建」）：

> 「捕获：…+ agent 主动出站（`post_gateway_send`，新建）。」

但实现层 `plugins/observability/ptg_capture/__init__.py:157` 的 `_on_pre_gateway_dispatch` docstring 写：

> 「post_gateway_send semantic capture is deferred to v2 **per D4**.」

ADR-V6-053 P2-T5 复核发现**三重假绿**：

1. **设计承诺**：ADR-V6-003:11 要求 post_gateway_send「新建」。
2. **不存在的决策借口**：docstring 声称「per D4」——但 **ADR-V6-003 根本没有 D4 段**（只有 `## Decision`，无 `### D1/D2/D3/D4` 子段；grep `^#+\s*D4` 在 ADR-V6-003 零命中）。D4 是凭空引用。
3. **零实现**：grep 全仓 `post_gateway_send` 仅命中 ptg_capture:157 docstring 一处；gateway 发送链路（`gateway/run.py`、`gateway/relay/adapter.py`）**无任何 emit / hook 注册 / handler**。这个「捕获点」从未被 emit、从未被消费。

这是比 citation（P2-1，至少写了无消费）更严重的假绿：**设计承诺 + 用不存在的决策掩盖未实现 + 零落地**。正是「别假绿骗我、严重怀疑偷工减料」担忧的形态——用看起来正式的「per D4」掩盖「没做」。

## 决策

### D1：正式化 defer（不在本批真建），但补**真** ADR + 真理由

post_gateway_send **继续 defer**（不在 P2 批次真建），但把「假 D4 借口」替换为**真 ADR（本 ADR）+ 真理由**。defer 的真理由：

- **pre_gateway_dispatch 已覆盖 Phase 0 出站审计**：`_on_pre_gateway_dispatch`（ptg_capture:154-164）在出站消息 auth/dispatch **前**记审计日志（`PTG capture [outbound]`），Phase 0 的出站可观测性已有落脚点。
- **post_gateway_send 的增量是「发送后语义捕获」**（成功/失败/送达语义），需要在 gateway 发送**完成**处挂钩回调——这是 gateway 核心发送链路的行为变更 + observability→gateway 跨层集成，超出 Phase 0（ptg_capture 黄金挂钩点：post_tool_call + pre_gateway_dispatch）范围。
- **真建需另起实施 ADR + gateway 层完成回调设计**，属 Phase 1+ 范围，不在 P2（可观测性 + 措辞校准）批次。

### D2：ptg_capture docstring 去假 D4，指向真 ADR

`_on_pre_gateway_dispatch` docstring 改：

- 旧：「post_gateway_send semantic capture is deferred to v2 **per D4**.」
- 新：「post_gateway_send semantic capture is deferred **per ADR-V6-064**：pre_gateway_dispatch 已覆盖 Phase 0 出站审计；post-send 语义捕获需 gateway send-completion 回调（跨层行为变更，Phase 0 外）。**替代先前无依据的 'per D4' 引用——ADR-V6-003 无 D4 段**（T5 审计）。」

这把「不诚实的 defer」（假 D4）变成「诚实的 defer」（真 ADR + 真理由 + 显式承认原借口无依据）。

### D3：测试钉死诚实性（C4）

`tests/plugins/observability/test_adr_v6_064_post_gateway_send_defer.py`：

- `test_docstring_references_real_adr_not_phantom_d4`：ptg_capture 源码含「ADR-V6-064」+ **不含**裸「per D4」（假借口清除）。
- `test_post_gateway_send_has_no_emit_or_handler`：grep agent/gateway/plugins 全 .py，断言无 `emit/register/subscribe/on("post_gateway_send"`——确认 defer 状态诚实（若未来真建，测试红提醒更新本 ADR）。
- `test_adr_v6_003_has_no_d4_section`：钉死「ADR-V6-003 无 D4 段」事实——原「per D4」借口的无依据性被永久记录。
- `test_adr_v6_064_exists_and_states_rationale`：本 ADR 存在 + 含真理由（pre_gateway_dispatch + defer）。

## 不做（诚实边界）

- **不**在本批真建 post_gateway_send——gateway send-completion 回调是跨层行为变更（observability→gateway），需独立实施 ADR + gateway 层设计，属 Phase 1+ 范围。P2 批次（可观测性 + 措辞校准）不引入核心发送链路行为变更。
- **不**声称 post_gateway_send「已支持」或「v2 会自动到」——本 ADR 明示「defer + 需另起实施 ADR」，不画饼。
- **不**删 ADR-V6-003:11 的 post_gateway_send 设计——它是合理的未来捕获点设计；本 ADR 只补「为何现在 defer」的真决策，不改设计意图。
- **不**改 `_on_pre_gateway_dispatch` 的 Phase 0 行为（allow + 审计日志）——它工作正常，只改 docstring 的假 D4 引用。

## 验证

- 0 行为变更 / 0 schema 改动 / 0 hook 接线 → 无 ruff 功能影响。
- 新测试（4 例）本地绿 + ruff 净。
- 提交后 CI 三连核验。
