# ADR-V6-035：match_atom 等价组治理面（action 20）

- **状态**：Accepted（Phase 1b 收口·批次 3，2026-07-20）
- **日期**：2026-07-20
- **Phase**：1b（诚实化）
- **关联**：ADR-V6-022（母纲，桶 C / action 20）、ADR-V6-026（精度⑦ 证死，recall-only 门禁）、ADR-201（V5 match_atom 源）
- **铁律**：C1（匹配策略即决策，版本化）、C4（等价组回归测试）、C3（生产现实——策略须可审计非黑盒字面量）

---

## 背景

ADR-V6-022 桶 C / action 20：「样本扩充（真实用户分布）+ match_atom 等价组偏松治理」。

- **样本扩充** = 桶 D（需真实用户 memo 扩样本，median 18 字单句 vs 生产多句口语；门禁过≠生产过）—— **诚实延迟**，上线后由真实使用填补，本 ADR 不触碰。
- **match_atom 等价组偏松治理** = 本 ADR 范围。

母纲审计（镜头 8）+ Explore 亲核（91 tool-use）发现 match_atom 等价组是**eval 的匹配策略**（决定哪些 predicted/expected 对算同一原子，直接移动 recall/precision 数字），但治理面全无：
- `_STATE_EQUIVALENT`（R1，3 对）+ `_R7_EQUIVALENT`（R7，4 对）是 **inline 在 match_atom 函数体里的字面量**——无版本、无审计、改了无记录。
- `_PERSON_SYNONYM_GROUPS`（R3，6 组）虽在模块级但无版本/描述。
- 文本重叠阈值 `0.4` 在 `_text_overlap` 默认 + R2 inline **重复硬编码**两处。
- 无 `describe()` / 无 eval 报告披露——策略是黑盒。

## 决策

### D1：治理化（零行为变更）

重构 `tests/benchmark/match_atom.py`，把匹配策略升级为**受治理、版本化、可审计**的面，**值零变更**（现有 `test_match_atom.py` C4 锁全绿证零回归）：

1. **inline 字面量上提为模块级命名常量**：`_STATE_EQUIVALENT` → `R1_STATE_EQUIVALENT_GROUPS`（frozenset of frozenset，runtime 不可变）；`_R7_EQUIVALENT` → `R7_INTENT_EQUIVALENT_GROUPS`。match_atom 函数体引用改名，逻辑逐字不变。
2. **阈值单点化**：`TEXT_OVERLAP_THRESHOLD = 0.4` 常量，`_text_overlap` 默认 + R2 inline 同引一处（消除重复硬编码）。
3. **版本化**：`MATCH_ATOM_VERSION = 1`——任何等价组/阈值改动须 bump（C1：匹配策略即决策）。
4. **可观测**：`describe_equivalences() -> dict` 返回结构化策略快照（version + 4 组集合 + 阈值，JSON 可序列化），供 run_eval / eval 报告披露当前匹配策略（不再黑盒）。

### D2：「偏松」是**有意的、保留的**（非治理目标改成收紧）

明确记录：等价组的宽松是 **design intent**，本 ADR **不收紧**。理由（实证）：
- 等价组编码「模型语义对了、标签表层形式不同」（老妈↔妈妈、开心↔高兴 valence 同、Consumption↔Evaluation 意图近）——**去掉它们 recall 假性塌缩**（V5 ADR-201 实证 R3 recall 在加 synonym 组前人为偏低）。
- 收紧等价组会 regress recall **而不助 precision**——precision⑦ 已证死（TP/FP confidence 重叠不可分，结构天花板 ~70%，ADR-V6-026），recall 是唯一门禁。
- 故「偏松治理」= **可见性 + 版本化 + 诚实文档化**，非收紧。若未来要收紧某组，须：改本 ADR + bump version + 改 governance 测试期望 + 重跑评测证 recall 不塌。

### D3：治理测试钉死策略成员

新增 `tests/benchmark/test_match_atom_governance.py`（10 测）：
- **PolicyIsGoverned**：version pinned + 等价组是模块级 frozenset（runtime 不可变）。
- **PolicyMembershipPinned**：6 R3 组 / 3 R1 对 / 4 R7 对 / 阈值 0.4 **逐条钉死**——改任一项此测红，强制 bump version + review。
- **DescribeEquivalences**：快照键完整 + 与常量一致 + JSON 可序列化（eval 报告可用）。

## 不做（诚实）

- **不收紧等价组**（D2 论证：regress recall 不助 precision，⑦ 证死）。
- **不扩样本**（桶 D，需真实用户，延迟）。
- **不改 run_eval 主流程**（match_atom 签名零变，run_eval 无需改；`describe_equivalences` 是可选披露面，run_eval 接入留后续）。

## 后果

- ADR-V6-022 action 20 的 match_atom 治理半**收口**（样本扩充半诚实延迟桶 D）。
- 匹配策略从「inline 字面量黑盒」升级为「命名常量 + 版本 + describe() + 成员钉死测试」——任何策略改动是显式、可审计、受测的决策。
- 反假绿：eval 数字背后的匹配策略可披露（`describe_equivalences()`），不再藏在函数体里；⑦ 决策（不收紧）显式记录防未来误收紧。
