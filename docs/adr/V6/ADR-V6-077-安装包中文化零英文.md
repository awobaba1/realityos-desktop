# ADR-V6-077：安装包与界面全量中文化（零英文）

- **状态**：已采纳
- **日期**：2026-07-21
- **决策者**：创始人
- **相关**：[[ADR-V6-076]]、[[ADR-V6-037]]、[[ADR-V6-038]]

## 背景（Reality）

创始人指令「安装包不要出现任何英文」——面向中文用户的桌面分发物（mac dmg / win exe 安装向导 + 应用启动后全部界面）必须做到用户可见处零英文。

现实调查（基于代码，非假设）：

1. **i18n 默认语言** `apps/desktop/src/i18n/languages.ts` `DEFAULT_LOCALE = 'en'`——新用户无 config 时 fallback 到英文，界面默认英文。
2. **i18n 中文 catalog 已完整**：`zh.ts` 2849 行 > `en.ts` 2704 行，走 `t()` 的 UI 中文键全覆盖，无需补译；问题只在「绕过 i18n 的硬编码英文」与「默认 fallback 是 en」。
3. **主进程硬编码英文** `electron/main.ts`：应用菜单（文件/编辑/视图/窗口/帮助）、role 子菜单、图像/链接右键菜单、dialog title（保存图像/登录云端/选择目录）、启动引导状态。
4. **渲染层硬编码英文**：placeholder（预设/搜索主题/消息）、aria-label（配置文件/移除引用/时间轴/更多操作）、路由 pane 标题（工件/消息/能力）、onboarding provider 描述（OAuth/API Key）、宠物生成不可用卡片、网关错误 toast title、HTTP IPC 错误响应体。
5. **安装向导配置** `package.json` build 字段：`dmg.title = "Install RealityOS"`（英文）、mac 权限描述英文、linux synopsis 英文、NSIS 未指定语言（跟随 Windows 系统语言，英文系统则英文向导）。

## 决策（Decision）

### D1：默认语言改为简体中文
`DEFAULT_LOCALE: Locale = 'zh'`。新用户无 config 时 fallback 到 zh，界面自动中文；i18n catalog 的 zh.ts 已完整覆盖。

### D2：硬编码英文 UI 直接改中文（绕过 i18n 的部分）
- 主进程 `electron/main.ts`：菜单/role/右键/dialog title/启动状态/HTTP body 全部中文。
- 渲染层：placeholder/aria-label/路由标题/provider 描述/宠物生成卡片/网关错误 toast 全部中文。

### D3：安装向导配置中文化
- `dmg.title` → `安装 RealityOS`
- mac `NSAudioCaptureUsageDescription` / `NSMicrophoneUsageDescription` → 中文
- linux `synopsis` → 中文
- NSIS `installerLanguages: ["zh_CN"]` → 强制单语言简体中文安装向导（无论 Windows 系统语言）。源码确认（`app-builder-lib/.../nsisLang.js:36`）：`zh_CN` → NSIS `SimpChinese` 语言文件，单语言时不显示语言选择器。

### D4：合理保留英文（非用户文案）
- **品牌专有名词**（i18n 惯例不译）：RealityOS / Hermes / Nous Portal / OpenAI / ChatGPT / Anthropic / MiniMax / Qwen Code / xAI Grok / OpenRouter / DeepSeek / GLM。
- **技术示例**（改了反而误导）：gateway URL placeholder（`https://gateway.example.com/hermes`）、profile id placeholder（`my-profile`）。
- **非用户可见**：代码注释、IPC `purpose:` 标识符（`'Media stream'` 等仅日志/调试）。

## 后果（Consequences）

- ✅ 中文用户开箱零英文（默认 zh + 硬编码全中文 + 安装向导强制中文）。
- ✅ 非 en 系统的英文 fallback 路径语义清晰：缺失键/失败 → fallback 到 zh（而非过去的 en）。i18n 单元测试已同步更新反映该新行为（见 C4）。
- ⚠️ 英文母语用户需手动在设置切换到 English（LOCALE_OPTIONS 仍含 en，可选）。这是面向中文分发物的合理权衡。
- ⚠️ NSIS 强制单中文：英文 Windows 用户也看到中文向导。符合「安装包零英文」硬要求。

## 验证（Verification）

- `npm run typecheck` 全绿（tsc -p . + tsconfig.electron.json，exit 0）。
- i18n 单元测试 21/21 全绿（含 4 个 fallback 测试同步更新：`defaults to zh without config` / `falls back to zh when config loading fails` / `unsupported language → zh` / `falls back to zh when key unresolved`）。
- 最终 grep 残留：placeholder 英文 0（除技术示例）、aria-label 英文 0、main.ts dialog message 英文 0。
- mac dmg 本地 build 绿（exit 0，`RealityOS-0.18.3-mac-arm64.dmg` 132M，sha256 `02d80d7e...`，中文版已放桌面）。

## C4 回归（铁律）

`DEFAULT_LOCALE` 由 `en` 改 `zh` 改变了 4 个 fallback 测试的期望值与语义——测试同步更新为反映新行为（zh fallback），测试名一并纠正（不再写「falls back to English」误导）。这不是迎合代码改测试，而是测试反映「面向中文用户默认 zh」这一正确决策的新行为契约。

## 诚实声明

- mac dmg 已本地 build 中文版并放桌面（可立即分发）。
- win exe 中文版需经 CI（desktop-build win job）build；本次 commit/push 后 CI 产出含中文 NSIS 向导的 exe。本地无 Windows 环境，exe 中文向导的实际渲染待 CI 产物/用户验证（NSIS `SimpChinese` 是 electron-builder 内置语言文件，机制可靠，但「眼见为实」待发版后确认）。
