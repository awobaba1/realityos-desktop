# ADR-V6-061 — ADR-V6-042 F6 接线范围措辞纠错（quark/theory 未接 F6）

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-053](ADR-V6-053.md) P2-T2 / [ADR-V6-042](ADR-V6-042.md) / 铁律 C1（一切皆决策） / [ADR-V6-037](ADR-V6-037.md) 反假绿

## 背景（审计发现 042 接线范围 overclaim）

ADR-V6-053 P2-T2 复核 ADR-V6-042（sample-size 分级 confidence 门 F6）时，发现其「后果」段第 75 行**过度声称 F6 的接线范围**：

> 原文：「公共门**可用于**所有新 insight 生产者（K域 A2 / quark B1 / theory B2）—— 统一信任度地板」

措辞「可用于」虽是软语（available），但与「所有新 insight 生产者」+「**统一**信任度地板」并列，产生「K域 A2 / quark B1 / theory B2 三者已共同形成统一地板」的**已落地印象**。实证复核（grep `cap_confidence_by_atom_samples` + `grade_confidence_by_sample` 全仓调用点）显示：

| 生产者 | F6 是否真接 | 证据 |
| --- | --- | --- |
| K域 A2（`compute_k_correlations`） | ✅ 真接 | `k_correlation.py:161-163` `if sample_size < MIN_SAMPLE: continue` + `conf = grade_confidence_by_sample(sample_size)` |
| InsightReportService（`_base.py`） | ✅ 真接 | `_base.py:133` `cap_confidence_by_atom_samples(...)`（D2 接线点） |
| quark B1（`aggregation.py`） | ❌ **未接** | `aggregation.py:73,80,89,96` 透传 `q.confidence`（Quark 自身分数），无 F6 import |
| theory B2（`engine.py`） | ❌ **未接** | `engine.py:211,231` 硬编码 `conf = 0.25 if degraded else 0.5` / `confidence=0.3`，无 F6 import |

即：**F6 只接到 2 处（K域 A2 + InsightReportService），quark B1 / theory B2 从未 import、从未调用 F6**。042 把三者并列为「统一信任度地板」是 overclaim。

## 决策（D1：改措辞，不去强行真接）

### D1：选 B（改 ADR-V6-042 措辞）而非 A（强行真接 quark/theory）

改 [ADR-V6-042](ADR-V6-042.md) 第 75 行，明确区分「已接」vs「未接」，保留「可供未来统一」的诚实愿景 + 指明若未来要接须另起 ADR。

**为何不选 A（强行真接）**：
1. **语义不天然适配**：F6（`cap_confidence_by_atom_samples`）是**报告级**「多 kind 最弱者 cap」语义——它需要一个 `atom_counts`（按 kind 分桶的样本计数）来 cap 报告级 confidence。quark B1 的 confidence 是**单记录级**（每个 Quark 自身分数，`aggregation.py` 透传），theory B2 的 confidence 是**单 derivation 级**（`engine.py` 硬编码 0.25/0.5/0.3）。两者都没有「多 kind 桶计数」这一 F6 输入的前提。强行套 report-level cap 会扭曲其单记录语义。
2. **行为变更须另起 ADR**（C1）：真接要改 `aggregation.py:_aggregate_one` 与 `engine.py:_build_derivations` 的置信度生成路径，属行为变更，不是 042（文档勘误）能覆盖的范围。
3. **无证据 quark/theory 存在 F6 类假绿**：F6 修复的是「LLM 在 n=2 数据点写强结论被缓存为 0.8 高可信」（InsightReportService 的报告级问题）。quark/theory 的单记录置信度本就偏低（0.25~0.5），且不在「报告级缓存高可信」路径上，当前无 overclaim 的实证危害。

故本 ADR 范围 = **只纠措辞**（0 代码/0 测试改动），把「未来若要统一须另起 ADR」作为明确 backlog 钉下，而非现在做语义不适配的强行接线。

### D2：042 第 75 行新措辞

```
- 公共门已接到 K域 A2（`k_correlation.py:163`）与 InsightReportService（`_base.py:133`）；
  quark B1（`aggregation.py`）与 theory B2（`engine.py`）暂未接 F6（其单 record / 单 derivation
  置信度语义不天然适配 report-level cap，若未来需统一须另起 ADR）—— 见 ADR-V6-061
```

## 不做（诚实边界）

- **不**强行把 F6 接到 quark/theory——语义不适配（D1），且属行为变更须另起 ADR。
- **不**声称 quark/theory 当前存在 F6 类假绿——无实证（它们单记录置信度本就低，不在报告级缓存路径）。
- **不**改 042 的 D1/D2/D3（F6 公共门本身 + InsightReportService 接线 + 27 测试）——这些实证为真（K域 + _base.py 真接，集成测实证 cached confidence floor 到 0.1）。本 ADR 只纠「后果」段接线范围的 overclaim。
