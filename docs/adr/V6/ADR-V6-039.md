- **状态**：Accepted（Phase 2 启动·Batch 1 前，2026-07-20）
- **日期**：2026-07-20
- **Phase**：2（Quark 夸克提取 + Theory 理论推导，文本子集）
- **关联**：ADR-V6-032（Phase 2 IDL 契约）、ADR-V6-022（母纲·Phase 2 路线图排除）、ADR-V6-038（确认 Phase 2 为允许推进的下一步）、ADR-V6-011/031（Phase 2.5 声学降级）
- **铁律**：C1（本决策一 ADR + 每批实现配套测试）、C5（LLM 输出 schema gate + confidence 门控）、C6（LLM call log + prompt 版本化）、C7（失败 → DLQ）、C3（生产现实——核心验证需真实用户，诚实延迟）

## 背景

ADR-V6-022 母纲把 Phase 2 全集（ServiceOrchestrator/Quark/Theory/RAG/commitments/G-K-V）排除在「100% 全量完成」外，列为路线图 Phase 2。ADR-V6-038 收口 Phase 1b 后，用户「都做」推进三方向之一**就是 Phase 2**（ADR-V6-038 §背景）——故**启动 Phase 2 现在合规**，非提前偷跑。

ADR-V6-032 已钉死 Phase 2 IDL（`plugins/memory/ptg/phase2_contracts.py`：`QuarkExtractor` / `TheoryEngine` Protocol + `QuarkRecord` / `TheoryDerivation` pydantic + 常量），且 PTG Schema 零迁移（`tool_events.quark_evidence` 列 + `insight_aggregation` 表 + `relations.delta` 早建好）。本 ADR 定义**实现**边界——哪些能做（可自主 + 可单测）、哪些诚实延迟（需真实数据/被砍依赖）。

---

## 决策：实现 Quark + Theory 文本子集（LLM 近似），分 3 批

### 做（IDL 已钉 + 表已建 + LLM 近似可单测 = 真实现，非假绿）

#### Batch 1：`realityos_quark` 插件 — QuarkExtractor（3 文本 kind）

- 实现 `QuarkExtractor.extract(quark_evidence_rows, capture_text) -> list[QuarkRecord]`：从 memo `capture_text` + `tool_events.quark_evidence` 行为证据，用 LLM 提取 **Identity（人）/ Meaning（意图/任务）/ Feeling（情绪）** 三类文本 Quark（`PHASE2_QUARK_KINDS` 子集，`phase2_contracts.py:78`）。
- 聚合进 R-原子：照 `QUARK_TO_ATOM_MAP`（7 条映射已钉死，`phase2_contracts.py:108-116`）把 quark 聚合到 `identity_events`/`meaning_events`/`feeling_events`。
- C5/C6/C7：LLM 输出过 `QuarkRecord` pydantic + confidence 门控；`llm_call_logs` 记录；失败 → DLQ。
- 落点：`plugins/realityos_quark/{__init__.py, plugin.yaml, extractor.py, aggregation.py, prompt_templates/}`。照 `realityos_insights` 插件模板。

#### Batch 2：`realityos_theory` 插件 — TheoryEngine（LLM 近似 PC/FR 骨架）

- 实现 `TheoryEngine.derive(user_id, atoms, relations) -> list[TheoryDerivation]`：**LLM 近似**推导 PC（7 约束：Time/Energy/Cognition/Emotion/Social/Execution/Environment）+ FR（5 维：Career/Interpersonal/BodyMind/Learning/Finance）骨架（`phase2_contracts.py:157-159` 明示「Phase 2 = LLM approximation of derivation skeletons」）。
- 写 `insight_aggregation`：`constraint_state`（PC/PhaseTransition/PASCR/Conservation）+ `fr_snapshot`（FR）——**不可**用 `detected_pattern`（pydantic 拒，那是 Pattern-Discovery sink）。
- 单向数据流（§4.7 不变式）：Theory 层**只读** atoms/relations，**绝不回写**原子层。
- 落点：`plugins/realityos_theory/{__init__.py, plugin.yaml, engine.py, prompt_templates/}`。

#### Batch 3：调度接线 + UI 读取

- 消费链：`post_tool_call` hook（或调度任务）触发 quark 提取 → 聚合进原子；周期任务（startup-lazy 调度）跑 theory derive → 写 `insight_aggregation`。
- UI：洞察页读取 `constraint_state`/`fr_snapshot`（复用 `realityos_insights` 读取范式 + 冷启动门控）。
- 端点：`GET /api/insights/theory`（或并入既有 insights 端点）。

### 不做（诚实延迟——需真实数据/被砍依赖，做了是空中楼阁）

- **Theory 统计公式**（`PC-Energy = σ(R1.fatigue, R10.sleep_deviation)` 等）：`phase2_contracts.py:154-161` 明示 Phase 2.5。Phase 2 只允许 LLM 近似；`time_trend_daily` 表未建、无 R10 节律连续值来源。
- **Time / Behavior / Context / Network Quark kinds**：`phase2_contracts.py:69-72` 钉形状但标「防空跑」（依赖被砍的 Layer B/C 连续音频 / 多人场景 / SED）。
- **Phase 2.5 全部**（AcousticProvider SER / 韵律 / 声纹 / 分离）：ADR-V6-011/031 降级，「数月工程」。
- **桶 D 验证**（§8 Phase-Gate KR：Day-7 留存 >40% / 纠错率连续 2 周下降 / 回测准确率 >75%）：ADR-V6-022 §66-73 明示需外部用户上量。代码侧能验证**形状正确性**（单测），但**无法验证推导对真实用户准不准**——那是上线后真实使用填补。
- **Phase 2 其他模块**（Personal RAG / commitments / G-K-V 三域 / ServiceOrchestrator）：母纲路线图项，IDL 未钉；先做 Quark+Theory 子集更稳。

## 结构性约束（踩坑警示）

1. **具体类绝不入 `phase2_contracts.py`**：`test_only_protocols_and_pydantic_models_ship`（`tests/test_phase2_contracts.py:47-72`）守门——未来加具体实现类必红，强制走 ADR + `bump PHASE2_CONTRACT_VERSION`。实现类一律进新插件目录。
2. **不建 dormant 空插件目录**：ADR-V6-032 D3 明示「造空壳本身就是假绿」。要么真做实（Batch 1/2 落地时才建），要么不建。
3. **Theory 写 `insight_aggregation` 用 `constraint_state`/`fr_snapshot`**，不可 `detected_pattern`（pydantic 拒）。
4. **任何契约形状变更**（Quark kinds 增减、map 改、字段改）= `bump PHASE2_CONTRACT_VERSION` + 新 ADR（C1）。
5. **Theory 单向数据流**（§4.7）：derive 只读 atoms/relations，不回写。
