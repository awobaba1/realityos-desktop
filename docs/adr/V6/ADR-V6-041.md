# ADR-V6-041：.pyc 防御（conftest sys.dont_write_bytecode）— 反假绿单点故障修复

- **状态**：Accepted（2026-07-20）
- **日期**：2026-07-20
- **Phase**：2-A 前置（P0 反假绿基建）
- **关联**：ADR-V6-040（danao14 综合母纲 / P0 清单 F1）、ADR-V6-022（母纲）、strategy-02 T-0
- **铁律**：C1（本决策一 ADR + 配套测试）、C4（缺陷→回归测）、C7（无静默失败 — stale .pyc 是静默失败源）

---

## 背景

danao14 调研 strategy-02 T-0 判定：**`tests/conftest.py` 缺 `sys.dont_write_bytecode` 是整个可信度系统的单点故障**。7 轮调研实地核验确认：`tests/conftest.py`（897 行）确无该设置。

**为什么是单点故障**：
1. pytest 导入会写 `__pycache__/*.pyc` 进源码树
2. **stale .pyc 会被 import 代替已编辑的源码** —— 掩盖「代码改动未生效」的事实
3. 这是经典 C7 静默失败：「测试过但生产跑旧码」
4. CI 三连核验（headSha 内容指纹）必须比对真实源码,而非可能滞后的字节码

实地核验：清理前 `plugins/` 下有 **74 个遗留 `__pycache__`**（之前脏运行产物），正是 F1 警示的脏字节码风险实证。

---

## 决策

### D1：conftest.py 加 `sys.dont_write_bytecode = True`

`tests/conftest.py` 在 PROJECT_ROOT sys.path 设置后、per-file isolation 注释前,模块级加：

```python
sys.dont_write_bytecode = True
```

带反假绿注释（strategy-02 T-0 / C7 / CI 三连核验依据）。

**作用域**：conftest import 时（进程启动）设置,后续该测试进程的所有 import 跳过 .pyc 发射。subprocess-per-file runner（`scripts/run_tests_parallel.py`）每文件重新 import conftest → 全局一致。已导入模块的字节码不受影响 —— 意图是阻止**未来**测试 import 污染树。

### D2：C4 回归测 `tests/test_conftest_pyc_defense.py`（3 测）

1. `test_conftest_sets_dont_write_bytecode`：断言 `sys.dont_write_bytecode is True`
2. `test_importing_module_emits_no_pyc`：写临时 .py → importlib 导入 → 断言无 `__pycache__/`（功能实证）
3. `test_no_stale_pycache_in_source_tree`：环境守卫 —— `plugins/` 下无遗留 `__pycache__`（若脏运行遗留则大声红,提示清理）

### D3：清理 74 个遗留 `__pycache__`

清理前确认：`.gitignore` 已忽略 `__pycache__/`（行 11）+ `*.pyc*`（行 10）+ `git ls-files | grep __pycache__` = 0（无被跟踪）→ 清理安全。`find plugins -name __pycache__ | while read; rm -rf` 全清。

## 后果

- 环境守卫测防止未来脏运行遗留 __pycache__ —— 出现即大声红
- credibility 系统单点故障（strategy-02 T-0）消除
