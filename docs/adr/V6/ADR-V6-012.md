# ADR-V6-012：Phase 1a 全量收口——读回路 + 评测门禁 + 灾备 + 默认 provider + shutdown drain

- **状态**：Accepted（Phase 1a 生产级可分发定案；代码已落地 commit `93a317520`，测试/核验见文末）
- **日期**：2026-07-19
- **Phase**：Phase 1a（AI Life Memo）——把「能产原子」升级为「可召回 · 可评测 · 可恢复 · 零配置 · 不丢」
- **关联**：ADR-V6-011（心脏 Atomizer）、ADR-V6-008（PTG 数据层）、ADR-V6-010（分发）、ADR-V6-007（阈值下调，本 ADR ⑦ 实测后延迟）；V5：extract-quality-baseline（v11 基线）、run_eval/match_atom（评测资产）
- **铁律**：C1（本 ADR）、C2（软删+version，事件表已具备；备份只读不改 live）、C3（生产现实即真理——200 样本真实 LLM 量，非自判）、C4（每缺陷/契约→回归测试）、C5（评测量的是 post-gate 落库原子）、C7（备份反假绿 + drain 不静默丢 + DLQ 全路径）

## 背景

ADR-V6-011 落地心脏后，全量审计（fork recon）暴露 5 个让 Phase 1a **不可分发**的缺口：

1. **原子写了无人能读回**（最大缺口）：Atomizer 把 R0/R1/R2/R3/R7 写进事件表 + 物化进 entities/relations，但 PTGStore 对事件/实体/关系**零公开读 API**——结构化数据是 write-only，用户永远只能从原始 memo 文本（search_memos_fts）召回，抽出来的「张三/交报告/厦门」图谱永不可见。
2. **无评测门禁**：「提取质量」无量化，任何改动都靠自判——正是用户怒斥的「假绿」温床。
3. **无灾备**：V6 全量数据钉死本机一块盘，盘坏/失窃/误删 = 用户人生图谱全没。这是数据资产产品**永不许上线带**的单一失败。
4. **fresh install 零原子**：DEFAULT_CONFIG 无 `auxiliary.extraction`，call_llm → auto → 无 provider → 每 memo 必进 DLQ，零原子。
5. **shutdown 丢原子**：守护线程抽取途中 shutdown 关库 → 写 closed DB → 原子静默丢失（C2/C7）。

## 决策

### 决策 1：读回路——三个读 API + 图谱渲染器（§4.3A）

- **`PTGStore.recent_atoms(user_id, memo_id?, limit)`**：从 4 事件表**反向重建**原子 dict（meaning_event 的 `intent_class='Need_To_Do'`→R2，其余→R7；confidence 取 `relation_confidence` 优先）。这是评测与召回的公共底座。
- **`search_entities(user_id, query, limit)`**：图谱节点名 LIKE 召回（CJK 2 字名也命中，同 search_memos_fts 的 OR-join 思路）。
- **`relations_for_user(user_id, entity_id?, limit)`**：边查询带 JOIN 出 subject/object 名（渲染器无需二次查表）。
- **`recall.render_relations_block(store, user_id, query, token_budget=800)`**：把 self→X 边折叠成 markdown 进 prefetch。**类型权重**（§4.3A）：`interacts_with`(person)=3 > `has_task`(task)=2 > `mentions`(context/topic)=1；置信序次之；**硬 ≤800 token 上限**（`_estimate_tokens` 无依赖启发：CJK≈1 tok/字、Latin≈4 字/tok），超限截断非静默突破。空命中→空串（fresh/无关 turn 不注入噪声）。`provider.prefetch` 接通 memo recall + graph block 两段。

### 决策 2：评测门禁——量 post-gate 落库原子（§11.3，反假绿）

移植 V5 三资产（`match_atom` 类型感知匹配 + `samples_v0.jsonl` 200 样本 + `run_eval`），但**关键诚实改进**：V5 直接调 LLM 量 raw JSON 输出；V6 走**完整 Atomizer 管线**（抽取→C5 门→写事件表→图谱物化）后用 `recent_atoms(memo_id)` **读回真正落库的原子**再评分——量的是**用户真拿到的 post-gate 数**，非 raw LLM 输出。这是数据资产产品唯一诚实的「提取质量」单位。

**200 样本真实 DeepSeek-v4-flash 端到端（temperature=0，8 并发）**：

| 类型 | Exp | Pred | Match | Precision | Recall | 门禁 | 结果 |
|---|---|---|---|---|---|---|---|
| R3_Person | 122 | 129 | 112 | 86.8% | **91.8%** | ≥85% | ✅ |
| R2_Task | 93 | 136 | 86 | 63.2% | **92.5%** | ≥80% | ✅ |
| R1_SelfState | 79 | 124 | 70 | 56.5% | **88.6%** | ≥70% | ✅ |
| R7_Expression | 44 | 50 | 37 | 74.0% | **84.1%** | ≥60% | ✅ |
| **Overall** | | | | **65.6%** | **90.2%** | F1 | **76.0%** |

**Phase Gate（CLAUDE.md R3>85/R2>80/R1>70/R7>75 recall）全绿**，0 errors。`v6_eval_report.json` 为实证（反假绿）。C5 门（V5 阈值）**未过滤掉正确原子**（recall 90.2% 远超门禁）。

### 决策 3：灾备引擎（§6.9 三防线之①③）

- **`backup_ptg(store, dest, keep=30)`**：在线 `conn.backup()` 在 store lock 下与 atomize 守护线程串行；30 rolling 裁剪。
- **`verify_backup(path, expected_counts?)`**（防线③月演练核心，**反假绿契约**）：`SELECT 1` 探针（非库文件必抛）+ `PRAGMA integrity_check`（撕裂写/页损坏）+ 7 脊柱表**任一缺失（-1）无条件判红**（不只当有 live 数对比时）+ 与 live 对比（backup>live=损坏）。**损坏/缺表/缺核心表必红，绝不假绿**——这是用户「不要假绿」诉求的承重实现。
- **`restore_drill(store, path)`**：以 live 计数复核备份，月演练（restore→对账→失配告警），RPO≤24h RTO≤30min。第二地点 opt-in（永不强制云）。8 回归锁契约（含 corrupt-file/missing-table/over-count 三种假绿必红）。

### 决策 4：默认 provider——extraction 继承主模型（零额外配置）

`DEFAULT_CONFIG["auxiliary"]["extraction"] = {provider:"auto", model:""}`——auto+空 model 经 `_resolve_auto` Step1 继承**主对话模型**。用户跑现有 `hermes model` 选一次主模型（对话本就必选），atomization **零额外配置**生效。已验证 `_get_auxiliary_task_config("extraction")` 返回 `auto`。修 Explore 实证的硬阻塞。

### 决策 5：shutdown drain——在飞线程有界 join，防 closed-DB 丢原子

`_spawn_atomize` 把守护线程入 `self._atomize_threads`（死线程裁剪）；`shutdown` 先 null atomizer（停新调度），再**有界 join**（`shutdown_drain_timeout` 缺省 35s = LLM 超时+余量）在飞线程**后**才 close——在飞抽取写完原子再关库。卡死的有界不 hang（超时警告后照常关，记录「原子可能丢失」）。2 回归锁双契约（finishing→drain 原子落库；hung→有界不 hang）。

### 决策 7（REV-9/ADR-V6-007）：阈值下调——**诚实延迟**

ADR-V6-007 原拟下调 C5 阈值（R3≥0.5/R7·R8≥0.5/R2≥0.4），**门禁是 precision≥0.85**。决策 2 实测 **precision 65.6% 远低于门禁**——下调阈值只在 precision 已高时才安全（否则放更多 FP 进库）。**诚实延迟**：阈值维持 V5 原值（R3=0.8/R2=0.7/R1·R7=0.5），待 precision 改善（prompt v12 / 词汇表注入 ADR-049 / dedup）后再评估下调。**非假绿强行下调**。

## 后果

**正向**：Phase 1a 从「能产原子」升级为**生产级可分发**——图谱可召回（prefetch 注入）、质量可量化（200 样本全绿实证）、数据可恢复（备份+演练反假绿）、零配置（继承主模型）、不丢失（drain + 软删 + DLQ）。用户「100% 全量完成、生产级对外发布」的可验证承重件齐备。

**代价/风险**：
- 评测 cost：200 样本真实 LLM ~¥1.86/次全量跑（可 `--limit 50` 冒烟）；temp=0 确定性可复跑。
- `_estimate_tokens` 是无依赖启发（无 tiktoken）——仅用于截断 prefetch block，非计费；混合文本 ±20%。
- precision 65.6% 是已知债（R1 56.5% 最弱，模型过抽 self-state）——是 prompt/抽取侧问题，非门控问题；留治 v12/词汇表/dedup。

**明确不做（Phase 1a 之外，勿当已完成）**：
- ❌ CaptureSubscriber 语义 DB sink（tool/outbound 事件表）——Phase 2（D4），现仅 fire+log 验证接线（Phase 1a DoD=接线已足）。
- ❌ C5 阈值下调——precision 未达 0.85 门禁，诚实延迟（决策 7）。
- ❌ 备份防线②（音频冷 rsync）——Phase 1 无音频。
- ❌ 共现边 / R1·R7 图节点 / 拼音归一化 / 词汇表注入 / 声学 Phase 2.5——均同 ADR-V6-011 留治。
