# ADR-V6-003: 数据捕获边界——流经即捕获（REV-7 决策②）

- **Status**: Accepted
- **Date**: 2026-07-17
- **Refs**: V6 架构设计 §0.6 决策② / §0 L25 / §5.1 硬约束

## Context
操作电脑（要读写文件 / 执行命令 / 读屏）与 §5.1 硬约束（不做系统级监控、不读屏/剪贴板/其他 App）存在张力。需精确调和线。

## Decision
- **捕获**：用户**委托 agent 执行、流经 agent 自身**的任务（`post_tool_call` 黄金挂钩点：tool_name/args/result/全链路 id）+ 用户与 agent 的交流（`pre_gateway_dispatch`）+ agent 主动出站（`post_gateway_send`，新建）。
- **不捕获**：agent 不监控用户在其他 App / 屏幕 / 剪贴板 / 文件系统里的行为（§5.1 不变）。
- **computer-use 读屏归类**：读屏是**执行手段**（「当你的手需要看」），其图像 / 内容**不作为数据源沉淀进 PTG**——与「不偷窥屏」相容。
- **工程支撑**：`tools/approval.py`（用户确认门）+ `tools/computer_use/permissions.py`（权限层）已在底座，边界非空话。

## Consequences
- ✅ 边界清晰，「操作电脑吸取数据」与「不监控」相容。
- ⚠️ computer-use 执行时屏幕内容进 LLM context，「LLM 瞥过敏感屏（密码/聊天/财务）」不可完全归零 → Phase 0/1 方案：computer-use 默认关闭 + 逐次用户授权 + 读屏图像不持久化 + 敏感区 redact。
