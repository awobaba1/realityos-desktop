# ADR-V6-070 — write-only 三件套读面（hermes trail）

**状态**: Accepted
**日期**: 2026-07-21
**关联**: 第四轮全量反假绿审计（ADR-037 维度）/ [ADR-V6-037](ADR-V6-037.md)（做了没发）/ [ADR-V6-063](ADR-V6-063.md)（citation 读面，同型先驱）/ [ADR-V6-065](ADR-V6-065.md)（dlq 读面，同型先驱）/ [ADR-V6-045](ADR-V6-045.md)（deletion_log）/ [ADR-V6-028](ADR-V6-028.md)（calibration）

## 背景（第四轮审计：ADR-037「做了没发」三件套）

第四轮全量审计 ADR-037 维度发现三张「write-only-no-consumer」表——都有真实 producer 在写、都有 ADR 添加的读 API、都有 C4 测试钉死读 API——但**零 CLI/API/UI 消费**（做了没发，ADR-V6-037 最致命假绿类；与 ADR-063 给 citation 修过的、ADR-065 给 dlq 修过的同一病）：

| 表 | producer（写） | 读 API（零消费） | 后果 |
|---|---|---|---|
| `deletion_log` | `soft_delete` / `cascade_soft_delete` / `purge` inline INSERT（ADR-V6-045） | `list_deletion_log` | R12 sovereignty 审计承诺「可追溯·跨重启可查」仅纸面 |
| `tool_events` | `ptg_capture` 每次 post_tool_call（observability） | `recent_tool_events` | 工具执行捕获纯 write-only 遥测 |
| `quality_metrics` | `calibration`（ADR-V6-028） | `recent_quality_metrics` | 校准时序不可观测（ADR-063 给 citation 修过的同型缺口） |

附 P2：`log_deletion` 公共方法零调用（三处 producer 全 inline INSERT 绕过它）；`dlq_resolve_all` CLI 未暴露 `--resolve-all`。本 ADR 聚焦 P1 三件套。

## 决策

### D1：单一伞命令 `hermes trail` 消费三表

新增 `hermes trail`（`hermes_cli/trail_cmd.py` + `hermes_cli/subcommands/trail.py`）作为三表的**唯一消费方**。设计取舍：

- **伞命令而非 3 个子命令**：三件套同批审计发现、同性质（write-only 观察面）、同口径修复；一个连贯读面 + 最小 main.py 接线（4 点 × 1，非 × 3）。`--type {deletion,tool,quality}` 选表读明细，每 type 独立可达 + 独立测试。
- **镜像 `hermes dlq`/`hermes citation`/`hermes k`**：thin delegate（store open → read → close），read-only（架构 §4.7，永不写永不改）。
- **概览模式**（缺省）：三表各读小窗口（3 条），一眼见哪张表有数据；`--type X` 进明细（默认 20，夹 [0,200]）。
- **诚实空状态**：三表均空时打「三表均空：尚未产生记录」+ 写入触发说明，绝不造假「一切正常」。

### D2：4 点接线（main.py）

| 点 | 位置 | 内容 |
|---|---|---|
| import | `main.py:331` | `from hermes_cli.subcommands.trail import build_trail_parser` |
| builtin | `_BUILTIN_SUBCOMMANDS` | `"trail"`（守卫 `test_builtin_set_covers_every_registered_subcommand`） |
| delegate | `cmd_trail(args)` | thin delegate → `hermes_cli.trail_cmd.cmd_trail` |
| parser | `build_trail_parser(subparsers, cmd_trail=cmd_trail)` | — |

运行时实证：`hermes trail --help` 输出含 `--type {deletion,tool,quality}` + 帮助文案。

### D3：C4 测试钉死可达性 + 反假绿守卫

`tests/plugins/memory/test_adr_v6_070_trail_read_surface.py`（9 测试）：
- **可达性 x4**：`"trail" in _BUILTIN_SUBCOMMANDS` + 每个 `--type` 真读出 seeded 行（deletion/tool/quality 各一）+ 概览读全三表。
- **诚实空状态 x2**：空概览打「最近 0 条」非造假 + 无 founder 打诚实提示。
- **静态反假绿守卫 x2**：`trail_cmd.py` 必须引用三个读 API 名（防 consumer 被静默拆线）+ `main.py` 必须含 4 点接线（import/parser build/cmd def）。

## 反假绿自检

- 三表 producer → 读 API → CLI consumer 全链通（seed→read→assert 双向实证）。
- 概览空状态诚实（不造「全绿」假象）。
- consumer 可达性有运行时测试 + 静态守卫双保险（防 ADR-037 复发）。
- read-only：cmd 全程零写入零修改（架构 §4.7 单向）。
