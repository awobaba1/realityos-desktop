# ADR-V6-063 — citation 可信度计数器读面（兑现 ADR-V6-043 可查承诺）

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-053](ADR-V6-053.md) P2-T1 / [ADR-V6-043](ADR-V6-043.md) / 铁律 C7（无静默失败 — 写了无消费是静默） / [ADR-V6-037](ADR-V6-037.md) 反假绿 / [ADR-V6-056](ADR-V6-056.md)（同类缺口先例：`hermes k`）

## 背景（审计发现 citation 计数器写了无消费）

ADR-V6-043（G1 可信度门）的「后果」段白纸黑字承诺：

> 「计数器可追溯 ✅ 跨重启可查可信度漂移」

但 ADR-V6-053 P2-T1 复核发现：生产代码里 `PTGProvider._observe_citation_quality`（`plugins/memory/ptg/provider.py:430/441`）**每轮**把 `citation_grounded_turns` / `citation_ungrounded_turns` 写入 SQLite `ptg_meta` 表，**全仓无任何消费方**——无 CLI 子命令、无 `hermes doctor` 检查、无 API、无 UI 读它。

这是 ADR-V6-037「做了没发」最致命假绿类别（与 ADR-V6-056 修复的 `compute_k_correlations` 同类）：承诺「可查」，实际是**无负载数据库写入**——计数器永远累加但没人看，G1 可信度漂移信号被收集后丢弃。ADR-V6-043 的「✅」是 paper-only。

## 决策

### D1：PTGStore 加 `citation_stats()` 读 API（store 层封装）

`plugins/memory/ptg/store.py` 新增 `citation_stats() -> {grounded, ungrounded, total, ungrounded_ratio}`，仿既有 `founder_user_id()`（同是 ptg_meta 读取，`with self._lock` + `self._conn.execute`）。read-only，never raises（C7），malformed value 计 0（防御手编辑 DB 行）。

**为何放 store 层**：`_conn`/`_lock` 是 PTGStore 私有；读取 API 应由 store 封装（分层铁律：upper 不直碰 lower 私有）。`_bump_meta`（写）与 `citation_stats`（读）现在同在 PTGStore，schema 对齐有单一归属。

### D2：`hermes citation` CLI 子命令（薄委托 citation_cmd）

新建 `hermes_cli/citation_cmd.py`（I/O adapter，仿 `k_cmd.py`）：`load_ptg_config` → `PTGStore` → `store.citation_stats()` → try/finally `close`（C7 teardown）→ 渲染。

- 默认：人类可读表（grounded / ungrounded / 无据占比 / 软 ⚠️ 提示当占比 ≥30%）+ 诚实空状态（无数据时显示「尚无观测」而非伪造 0）。
- `--json`：机器可读 JSON（未来 dashboard / doctor 消费）。

新建 `hermes_cli/subcommands/citation.py`（parser builder，仿 `calibrate.py`/`k.py`，handler 注入 `cmd_citation`）。

`main.py` 的 `cmd_citation` 是**薄委托**（`from hermes_cli.citation_cmd import cmd_citation as _run; return _run(args)`）——业务逻辑不进 `main.py` 巨型文件（与 `cmd_k` 同模式）。

### D3：接线 + `_BUILTIN_SUBCOMMANDS` 同步

`main.py` 四处接线（P1 教训：加子命令必须同步白名单，否则 `test_builtin_set_covers_every_registered_subcommand` CI slice 失败）：
1. `from hermes_cli.subcommands.citation import build_citation_parser`（import 段）
2. `def cmd_citation(args)`（薄委托，`cmd_k` 旁）
3. `build_citation_parser(subparsers, cmd_citation=cmd_citation)`（注册段）
4. `"citation"` 加入 `_BUILTIN_SUBCOMMANDS`（白名单）

## 不做（诚实边界）

- **不**删除既有 `_observe_citation_quality` 写入——G1 可信度信号有未来价值（ADR-V6-043 明示这是未来硬执行门的观察基础）。删写入 = 丢信号，方向错。本 ADR 只补读面。
- **不**加硬执行门（refuse-to-render ungrounded）——那需要 agent-loop answer hook，是行为变更，ADR-V6-043 标记为 next iteration，超出本 ADR（只补读面）。CLI 渲染里的 ⚠️ 是**软提示**，不是门。
- **不**改 `_observe_citation_quality` / `_bump_meta` 逻辑——只读它们写的表。
- **不**声称修复了 G1 幻觉——本 ADR 只让 G1 漂移**可观测**（兑现「可查」承诺），不阻断。观测是执行的前提，非执行本身。

## 验证

- `PTGStore.citation_stats` 单元测（`test_adr_v6_063_citation_stats.py::TestCitationStatsStore`，5 例）：空表 / 双计数器 / malformed 计 0 / 仅 grounded ratio=0 / 与 `_bump_meta` 写入模式 round-trip（无 schema drift）。
- `hermes citation` 渲染测（`TestCitationCmdRender`，4 例）：诚实空状态 / 渲染计数+占比+⚠️ / 阈值下不提示 / JSON 机器可读。handler 用 fake store（monkeypatch）隔离，不碰真实 DB。
- `_BUILTIN_SUBCOMMANDS` 含 `citation`：既有 `test_builtin_set_covers_every_registered_subcommand` 自动覆盖（注册即白名单）。
- `ruff check .`（全量）净。
- 提交后 CI 三连核验。
