- **Status**: Accepted
- **Date**: 2026-07-17
- **Refs**: CLAUDE.md 铁律 C1-C7 / REV-9 H7 / H10 / H11 / H12 / H14 / H15 / H16 / 决策 D1 / D3 / D5

## Context
V6 从 V5（FastAPI + PG + RLS 多租户）换底座到 hermes fork（Python + SQLite + Electron，单机单用户）。铁律 C1-C7 是项目宪法，必须在新底座下重新落地。REV-9 全量精读 + H11 spike 暴露 7 个落地缺口（H7/H10/H11/H12/H14/H15/H16）。

## 3. PG → SQLite 类型映射（H11 spike 结论，**0 阻塞**）

| V5 PG 用法 | V6 SQLite 替代 | 工作量 |
|---|---|---|
| `Vector(512)` pgvector | sqlite-vec `vec0` 虚拟表（见 §3a H16 约束） | M 🟡 |
| JSONB 字段（25+ 列） | TEXT + JSON1 `json_extract`；原生 SQL `->>` 改 `json_extract(col,'$.k')` | M 🟡 |
| GIN 索引（trigger_source） | 砍 / FTS5 | S 🟢 |
| HNSW / ivfflat 向量索引 | sqlite-vec 自带 KNN，PG 索引 DDL 丢 | S 🟢 |
| `pg_advisory_xact_lock` | `BEGIN IMMEDIATE` + `threading.Lock` | S 🟢 |
| `AT TIME ZONE 'Asia/Shanghai'`（7+ 处） | Python `zoneinfo` 本地化 + 冗余 `local_date` 列 | M 🟡 |
| `EXTRACT`/`date_trunc`（6 处） | `strftime('%w'/'%H')` 或 Python 算 | M 🟡 |
| `UUID(as_uuid=True)` + `gen_random_uuid()` | TEXT(36) + Python `uuid4`（无 server_default） | M 🟡 |
| `Numeric(p,s)` | REAL + `round` | S 🟢 |
| `DateTime(timezone=True)` | TEXT ISO8601 UTC | S 🟢 |
| RLS policy（14 表）+ `SET LOCAL bypass_rls`（30+ 处） | **单用户整体砍** | L 🟢 |
| Partial / 表达式索引 | SQLite 原生支持 | S 🟢 |
| `CHECK` / `UNIQUE` / `onupdate` | 原生支持 | XS 🟢 |

**未发现的 PG 专有构造（免迁移）**：无 `STDDEV/PERCENTILE/materialized view/触发器/EXCLUDE`（V5 baseline 全 Python 算）；无 DB 层全文检索（tsvector）；无 jsonb_agg 复合聚合。**预估 6-10 工日，0 阻塞。**

### 3a. H16 sqlite-vec 加载约束（新，spike 实测）
sqlite-vec 扩展跨三平台 wheel 齐全（macOS arm64/x64、Linux、Windows），**但 macOS 系统 Python 3.9 / 系统 sqlite3 CLI 没编 `--enable-loadable-sqlite-extensions`**，`load_extension` 属性不存在。V6 Python runtime 必须：
- 用 **python.org 官方 Python**（编译时开 load_extension），或
- 打包**自带开了 load_extension 的 Python**，或
- **兜底**：向量存 BLOB + 应用层 numpy 余弦（单机 <10k memo，暴力扫描 <50ms 可接受，换可移植性）。

Electron 侧 `better-sqlite3` 原生支持 `loadExtension`。**P0-4 schema 实现前须在 realityos-desktop 验证 Node/Python 路径打通**（向量 RAG + 声纹落库的前置门控）。

## 4. user_id 归一（H15）
V5 多租户，V6 单机单用户。
- 迁入 `user_id` 统一为 **founder 固定值**（如 `00000000-0000-0000-0000-000000000001`）。
- 保留原 V5 `user_id` 为 `v5_user_id` 迁移审计列（追溯来源，C6）。

## 5. location_log 例外（H10，D3 决策）
- V6 砍 GPS（桌面端无位置源），`location_log` **不建不迁**。
- 创始人 V5 PG dump **永久异地备份**（不在主库但数据未丢）。
- **不违 C2**：备份在即 = 数据未丢；产品决策（无 GPS 场景）显式记录于此。

## 6. V5 智慧继承清单（H12，防重踩）
V5 踩坑沉淀、V6 文档未提、须主动继承：
1. **专有名词 5 层防护**（V5 ADR-048~052）：热词 + 纠错字典 + LLM 注入 + R0_Entity + 拼音归一化 → `realityos_atomizer` 重做。
2. **内容安全 M-1**（V5 ADR-078）：fail-open 审核（标记不阻断）→ V6 保留，且因 computer-use 读屏，补 **M-2** 覆盖 agent 瞥过的敏感屏。
3. **ASR 纠错自学习**（V5 ADR-056）：编辑源文本 → diff 学习 → 重新提取。
4. **用户反馈** 18 target_type（V5 ADR-089）。
5. **渐进式置信度**（V5 ADR-045）。
6. **advisory lock** → `BEGIN IMMEDIATE`（§3）。

## 7. 入库阈值下调依据（D5 决策）
V6 阈值较 V5 整体下调（R3≥0.5 / R7·R8≥0.5 / R2≥0.4，V5 为 R3>0.8 / R2>0.7 / R0>0.7 / R1·R7>0.5）。依据：桌面端数据密度低于手机语音 memo（无被动语音流、无 GPS/设备信号源），下调换召回率防漏抽。**门禁**：跑评测基准（§11.3），precision <0.85 回调至 V5 阈值。详见架构设计 §4.3D 阈值表。
