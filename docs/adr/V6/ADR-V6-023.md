# ADR-V6-023：Sovereignty 接线 + 未成年人模式 Atomizer 集成（桶 B1+B2 收口）

- **状态**：Accepted（批次 1 旗舰项，2026-07-19）
- **日期**：2026-07-19
- **Phase**：1b（反假绿收口）
- **关联**：ADR-V6-022（母纲）、ADR-V6-020（复用 `_open_ptg_store_for_insights`+`_resolve_insight_founder` helper）、ADR-V6-021（复用贡献式 UI 注册）
- **铁律**：C1（本 ADR）、C2（cascade 只 soft-mark，无 hard DELETE——purge 另议）、C5（不涉——读原语已落库数据）、C7（全路径 fail-open，200+结构化载荷绝不向 UI 抛）、§6 主权层 + §6.7 PIPL §31

---

## 背景

ADR-V6-022 审计揭露桶 B 两条头号假绿：

- **B1 sovereignty 完全孤儿**：`plugins/realityos_sovereignty/sovereignty.py` 四个练习原语（cascade_soft_delete / export_user_data_json / set_minor_mode+is_minor / set_consent_tag+get_consent_summary）真实、测绿，但 `apps/desktop/src` 与 `hermes_cli/web_server.py` **零非测试调用**。`register()` docstring 自承「UI wiring is the next step」。**「数据主权」是 V6 招牌，用户实际摸不到任何主权按钮** = 卖点假绿。
- **B2 未成年人模式 Atomizer 不读**：`set_minor_mode`/`is_minor` 把 flag 落 `ptg_meta`，但 `atomizer.py`（637 行）全文零 `minor` 命中——flag 翻了 Atomizer 照抽 R1SelfState/R9Emotion 生物标记原子。**对外宣称「支持未成年人模式」= PIPL §31 合规风险**。

本 ADR 收口这两条：把原语接到用户面（后端 4 端点 + 桌面 UI），并让 Atomizer 真读 flag。

## 决策

### 1. 后端：`/api/sovereignty/*` 四端点（`hermes_cli/web_server.py`）

复用 ADR-V6-020 验证过的 `_open_ptg_store_for_insights()`（开共享 PTGStore，refcounted）+ `_resolve_insight_founder(store)`（解析单 founder tenant）。**零新 helper、零 churn 到通过的 insight/memory 测试**。

| 端点 | 方法 | 入参 | 调用原语 | 返回 |
|---|---|---|---|---|
| `/api/sovereignty/export` | GET | — | `export_user_data_json` | `{"status":"ok","data":{...}}`（整个 tenant JSON） |
| `/api/sovereignty/delete` | POST | `{mode:'A'\|'B', since?, until?}` | `cascade_soft_delete` | `{"status":"ok","marked":{table:count}}` |
| `/api/sovereignty/minor` | GET | — | `is_minor` | `{"status":"ok","enabled":bool}` |
| `/api/sovereignty/minor` | PUT | `{enabled:bool}` | `set_minor_mode` | `{"status":"ok","enabled":bool}` |

**fail-open（C7）**：store 开失败/无 founder/任何异常 → `{"status":"error","message":...}` HTTP 200，绝不 5xx、绝不向 UI 抛（与 ADR-V6-020/021 同语义）。

**删除是高危操作**：前端必须二次确认（dialog）；后端 `mode` 不在 {A,B} → 400 友好错误（非 500）。cascade 只 soft-mark（`deleted_at`），**不调 purge**（purge 是另一条显式 opt-in 面，本 ADR 不碰）。

### 2. Atomizer 未成年人 gate（`plugins/memory/ptg/atomizer.py`）

在原子物化路径读 `is_minor(store, user_id)`：
- `True` → **跳过 R1SelfStateAtom + R9EmotionAtom 的物化**（§6.7「敏感原子降级采集」），其余原子（R3/R2/R7/R8/R12/R0）正常。
- gate 落在物化层（不影响抽取本身——抽取仍跑，只是生物标记原子不落库），最小改动 + 零回归（pass1 v11 逐字不动）。
- 一次性读取：`atomize()` 入口读一次 flag 传下去，避免每原子查库。

### 3. 桌面 UI：Settings 页主权区（`apps/desktop/src/app/settings/`）

贡献式或直接在 Settings 加一个「数据主权」区块，4 控件：
- **导出我的数据**（按钮 → GET export → 下载 JSON；PIPL §45）
- **删除一段时间记忆**（Mode A/B 选择 + 时间窗 + 二次确认 → POST delete）
- **未成年人模式**（开关 → PUT minor；说明文案：开启后不再记录状态/情绪生物标记）
- （consent_tag 控件较细，本 ADR 先做前三；consent 留 follow-up，因 relations 浏览器尚未暴露行级 id）

i18n 4 locale 平价（TS 类型门禁，与 ADR-V6-021 同）。

## 关键实现点

### 为什么不接 consent_tag 控件
`relations` 行级 `id` 当前未暴露到任何 UI（记忆浏览器 relations Tab 只渲染边，无单条 id）。consent_tag 练习面需要 `relation_ids`，而用户还看不到单条 relation 的 id。**本 ADR 先做前三控件（导出/删除/未成年），consent_tag 控件推迟到「记忆浏览器 relations 行级 provenance」之后**（与 ADR-V6-021 推迟的行级 provenance 同根）。后端 `set_consent_tag`/`get_consent_summary` 原语已就位，只缺 UI。

### Atomizer gate 的最小落点
不在 `_extract_pass`（抽取层）改——那会动 v11/v12 prompt 语义，违 C6。在 `_materialize_graph`/`_write_atom` 路由层加 gate：抽取出的 R1/R9 原子，`is_minor=True` 时直接 drop（不落库，不进图谱）。抽取本身照跑（prompt 不动），只是生物标记原子被 gate 掉。零 prompt 回归风险。

### 删除的不可逆性边界
cascade_soft_delete 是 soft-mark（可恢复——`deleted_at` 可被清，虽无 UI 恢复面）。**真正物理 purge（`purge_soft_deleted`）本 ADR 不接线**——它是 §6.2 阶段2 显式 opt-in 面，需独立夜间 cron（ADR-V6-022 P2），不能让 UI 一键触发物理删除。前端文案必须诚实：「标记删除（可联系开发者恢复期 1 天内）」。

## 验证（反假绿）

- **后端**（`tests/hermes_cli/test_sovereignty_web_api.py`）：export 返回全表 JSON / delete Mode A 只动 memos / delete Mode B cascade 多表 / delete 坏 mode → 400 非 500 / minor GET+PUT 往返 / store 开失败 → error 非 500 / 无 founder → no_data 非 500。
- **Atomizer**（扩 `test_ptg_atomizer.py`）：`is_minor=True` 时 R1/R9 不落库（recent_atoms 不含）/ R3/R2/R7/R8/R12/R0 正常 / `is_minor=False` 零回归（全量基线不变）。
- **前端**（`SovereigntySection.test.tsx`）：4 控件渲染 / 删除二次确认 / 开关 PUT / 导出触发下载 / fail-open 错误态。
- **反孤儿核验**：`grep -rn "cascade_soft_delete\|export_user_data\|set_minor_mode" apps/desktop/ hermes_cli/web_server.py` 有非测试命中（不再是孤儿）。
- **三连核验**：headSha==CI 绿 + 内容指纹（`/api/sovereignty/`+`is_minor`+`SovereigntySection` 入包）+ 真实检查非 skip。
