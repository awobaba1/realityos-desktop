## 背景（第四轮审计：ADR-066 的 R4 收口不完整）

第四轮全量审计（5 维度 / 5 agent）复核 ADR-066 时发现：其 R4 收口**遗漏了 3 处直接兄弟**——同文件、同 observation surface、同「用户数据写入 + log-only + 无 DLQ」性质，同一审计口径本应一并修。另在 R6（测试诚实）维度发现 1 处与 ADR-068 同型的 Teknium 测试空桩。

| # | 位置 | 现状 | 与 ADR-066 的关系 |
|---|---|---|---|
| D1 | `store.py` `adjust_atom_confidence` UPDATE 失败 | warn + return 0 | **D3 insert_feedback 的亲兄弟**——同创始人校准信号（ADR-V6-028 §11.4）、同 `store.py`、同 observation surface 注释 |
| D2 | `store.py` `mark_task_outcome` UPDATE 失败 | warn + return dict | **D4 insert_tool_event 的亲兄弟**——同任务/事件捕获面 |
| D3 | `theory/__init__.py` `_persist_one` INSERT 失败 | warn + return False | **D2 re_extract_memo 的亲兄弟**——同「内层 LLM 已 DLQ、外层 DB 写不 DLQ」的 backstop 模式 |
| D4 | `tests/tools/test_file_operations.py` `_DeletedTestGitBaselineCheck` | 9 行 tombstone | **ADR-068 同型**——记 6 个测幽灵方法 `_check_git_baseline` 的死测试，keep-CI-green 考古债 |

C2 维度（DELETE FROM）第四轮复核仍干净；C7 心脏 bare-except-pass 仍全部合法。缺口集中在 ADR-066 的「收口完整性」与 R6 测试诚实。

## 决策

### D1-D3：补 best-effort DLQ（镜像 ADR-066 D3/D4/D2 已验证模式）

三处 except 均在 `with self._lock` / `with self.transaction()` **外**（方法级 try）→ lock/transaction 已释放回滚，重 acquire 写 `insert_dlq` **无死锁**。补 best-effort `self.insert_dlq` + fail-safe try（DLQ 写自身失败 → warn，observer 永不崩）。

- D1 `adjust_atom_confidence`：补 `source="store.adjust_atom_confidence", error_type="confidence_adjust_failed"`，`original_data` 含 `atom_type/atom_id/new_confidence/reason`。
- D2 `mark_task_outcome`：补 `source="store.mark_task_outcome", error_type="task_outcome_failed"`，`original_data` 含 `ref/outcome/atom_id/actor`。
- D3 `theory._persist_one`（模块函数，`store` 为入参）：补 `source="theory.persist_one", error_type="insight_persist_failed"`，`original_data` 含 `kind/name/aggregation_type/period_key/degraded`。

**诚实边界**：同 ADR-066 D3/D4——best-effort 自举限制（连接级失败时 `insert_dlq` 同 `_conn` 也失败 → fail-safe warning 兜底）；非连接级失败（SQL/schema 逻辑错）时 DLQ 能写。明确 fail-safe + 承认连接级失效，非假绿。

### D4：测试诚实化（删 tombstone + 补真覆盖）

`_check_git_baseline` **全仓零实现**（grep `plugins/ tools/ agent/ hermes_cli/` 0 命中）——该方法在本 fork 从未存在，tombstone 记的是 6 个测幽灵方法的死测试。按 ADR-038「不可验证不写」：

- **删** `_DeletedTestGitBaselineCheck` tombstone（记不存在代码的 ghost 类）。
- **补** `TestWriteFileDiagnosticSurface` 真测试：钉 `WriteResult.warning`/`lsp_diagnostics`（file_operations.py:176-190，真实的 git-workspace-aware 诊断面）round-trip + None 字段 `to_dict` 省略 + C4 守卫 `_check_git_baseline` 确实不存在（防半截复活静默 ship）。

### D5：C4 测试钉死

`tests/plugins/memory/test_adr_v6_069_dlq_brothers.py`（12 测试）：
- D1/D2/D3 各 3 测试：失败写 DLQ + 合法早返回不写 DLQ（边界）+ DLQ 自身失败不传播（fail-safe）。
- D1/D2/D3 各 1 静态守卫（wrap-lucky-green 教训）：whitespace-normalize 后断言 `source="..."` + `dlq write also failed` 存在，防换行侥幸绿。

## 反假绿自检

- 生产 bare `except: pass` = 0（D1-D3 全带 fail-safe try + warn）。
- 测试 pass-only 空桩 = 0（D4 tombstone 删除，换 4 条真断言）。
- 三处 except 均在 lock/transaction 外（无死锁）——已逐处核验缩进。
- 每处 DLQ 落地有运行时测试（drop table / monkeypatch 触发）+ 静态 source-string 守卫双保险。
