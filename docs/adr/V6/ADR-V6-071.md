# ADR-V6-071 — quark `llm_call_id` 断链修复（C6 可追溯）+ docstring 诚实化

**状态**: Accepted
**日期**: 2026-07-21
**关联**: 第四轮全量反假绿审计（C5/C6 维度）/ 铁律 C6（现实必须可重放）/ [ADR-V6-039](ADR-V6-039.md)（quark 引擎）/ [ADR-V6-049](ADR-V6-049.md)（quark B1）/ [ADR-V6-038](ADR-V6-038.md)（诚实化方法论）

## 背景（第四轮审计：C6 traceability 断链）

第四轮全量审计 C5/C6 维度发现 quark 管线一处 C6 断链 + 一处文档假绿：

| 位置 | 问题 | 铁律 |
|---|---|---|
| `extractor.py:82` | `records, _llm_id, _ok = self._llm(...)` —— LLM 调用生成了 `llm_call_id`（并写 `llm_call_logs`），但 `extract()` 用 `_llm_id` **丢弃**，不外传 | **C6**（每个事件必须带 llm_call_id） |
| `aggregation.py:78-102` | 写 identity/meaning/feeling 事件时不带 `llm_call_id` → 三事件行全 NULL | **C6** |
| `__init__.py:43` docstring | 承诺 `Returns {ok, extracted, aggregated, counts, llm_call_id}` —— 实际返回 dict（58-63 行）**根本没有 `llm_call_id` 键** = 文档假绿 | C1 诚实 |

后果：quark 衍生的原子事件无法追溯到产生它的 LLM 调用（C6 可重放性破坏）。LLM 调用本身已正确记日志（C6 ✓），断链只在「事件 ↔ 调用」的关联环节。

注：事件表（identity/meaning/feeling_events）本就有 `llm_call_id` 列，`insert_*_event` 的 `**kw` 也透传它 —— 基础设施齐备，缺的只是上层把 id 传下去。

## 决策

### D1：extract() 暴露 `_last_llm_call_id`（实例属性，不改 Protocol 签名）

`QuarkExtractor.extract()` 的 Protocol 签名是 `-> list[QuarkRecord]`（`phase2_contracts.py`），**不能改返回类型**（破坏契约）。改为存实例属性：

- `__init__` 初始化 `self._last_llm_call_id: Optional[str] = None`。
- `extract()` 把 `self._llm()` 返回的 id 存入 `self._last_llm_call_id`（不再 `_llm_id` 丢弃）。
- 空输入早返回时置 None（诚实：未调用 LLM → 无 id，非陈旧值）。

### D2：aggregation 接收 + 透传 llm_call_id 到三个事件 insert

`aggregate_quarks_to_atoms(..., llm_call_id: Optional[str] = None)` + `_aggregate_one(..., llm_call_id=...)`，三处 `insert_identity/meaning/feeling_event` 调用各加 `llm_call_id=llm_call_id`（走现有 `**kw` → `_insert_event` 透传链，落事件行）。

**向后兼容**：参数默认 None，非 quark 调用方不传则写 NULL（保持旧行为，不强制全局面传）。

### D3：`extract_and_aggregate` 兑现 docstring 承诺

`__init__.py` 读 `getattr(ext, "_last_llm_call_id", None)` → 传给 aggregation → 并入返回 dict（`"llm_call_id": llm_call_id`）。docstring 由「承诺但撒谎」变「承诺且兑现」，并注明 id 既透传到事件又返回给调用方。

### D4：C4 测试钉死全链路

`tests/plugins/test_adr_v6_071_quark_llm_trace.py`（7 测试）：
- **暴露**：`_last_llm_call_id` 与 `llm_call_logs.id` 一致 + 空输入 None + 静态守卫（旧 discard 绑定 `records, _llm_id, _ok =` 不复现）。
- **透传**：三事件表行各带 `llm_call_id == ext._last_llm_call_id`（非 NULL）+ 默认 None 写 NULL（向后兼容钉死）。
- **返回**：`extract_and_aggregate` 返回 dict 含 `llm_call_id`（兑现承诺）+ 空输入返回 None。

## 反假绿自检

- C6 全链路通：LLM 调用 → `llm_call_logs`（已有）→ 原子事件 `llm_call_id`（新透传）→ 返回调用方，四环实证。
- docstring 由假绿变真（承诺=兑现）。
- Protocol 签名未破坏（用实例属性而非改返回类型）。
- 向后兼容（默认 None，不强制非 quark 调用方面传）。
