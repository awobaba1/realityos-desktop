# ADR-V6-049 — Quark 提取层（Phase 2-B B1）

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-039](ADR-V6-039.md) Batch1 / [ADR-V6-040](ADR-V6-040.md) D4 Phase 2-B /
[ADR-V6-032](ADR-V6-032.md) Phase 2 IDL 契约 / 架构 §4.3E

## 背景（IDL 钉死，实现空）

ADR-V6-032 早在 Phase 0 就钉死了 Quark 层 IDL（`phase2_contracts.py`：`QuarkExtractor`
Protocol + `QuarkRecord` pydantic + `QUARK_TO_ATOM_MAP` 7 条映射 + `PHASE2_QUARK_KINDS`），
且 PTG Schema 零迁移（`tool_events.quark_evidence` 列早建）。但**实现空**——直到本 ADR，
没有任何具体 `QuarkExtractor` 实例化，`quark_evidence` 列无人写也无人读。ADR-V6-022 桶 C
item C4 指认的正是这个「接口定死、实现缺失」缺口。本 ADR 收口它。

## 决策

### D1：`realityos_quark` 插件 —— 具体实现，非空壳

`QuarkExtractorImpl`（`extractor.py`）满足 `QuarkExtractor` Protocol（`runtime_checkable`
过）。`extract(quark_evidence_rows, capture_text) -> list[QuarkRecord]` 经 LLM 提取 3 类文本
Quark。`aggregation.py` 按 `QUARK_TO_ATOM_MAP` **PRIMARY 映射**写原子。`extract_and_aggregate`
（`__init__.py`）是闭环节点。`register` 是 no-op（显式调用，非假装 live）。

### D2：只做 3 类文本 Quark（诚实范围）

`PHASE2_QUARK_KINDS = (Identity, Meaning, Feeling)`。其余 4 类（Time/Behavior/Context/Network）
在契约里**钉形状但标防空跑**（ADR-V6-039）：Time 走规则引擎非独立提取器；Behavior 依赖
post_tool_call hook / Phase 3 SED；Context 依赖 Phase 3 声学场景；Network 依赖 Phase 2.5+
多人。**Phase 2 用文本提取它们 = 编造**，故 extractor 显式 drop 非 Phase-2 kind 的记录
（`_parse` 里 `rec.kind not in PHASE2_QUARK_KINDS → continue`），绝不放行。

### D3：PRIMARY 映射，不造 aux R1（反假绿关键）

`QUARK_TO_ATOM_MAP` 每类有主/辅原子（如 Identity→R3 主/R1 辅）。本实现**只写主原子**：

| Quark | 主原子 | 落点 |
|---|---|---|
| Identity | R3_Person | `identity_events.person_name=value` |
| Meaning | R7_Expression | `meaning_events` atom_kind='R7'，task_description=value |
| Feeling | R9_Emotion | `feeling_events` atom_kind='R9'，trigger_source.trigger=value |

**辅 R1（SelfState）刻意不做**：R1 需要声学导出的 intensity，契约明示 Feeling 是「文本弱版」。
从每次文本提及造一个 self-state 原子 = ADR-V6-039 警告的噪声。aux R1 留 Phase 2.5 声学。

**Meaning 用 R7 不用 R2**：Quark 是信号（「看到『要交报告』」），不是任务。给信号盖
task_status/deadline = 编造生命周期。信号原样存 task_description，真任务由 atomizer/A3 显式
提升。诚实不丢信息。

### D4：C5/C6/C7 全闸（同 insights rails）

- **C5**：每条记录过 `QuarkRecord` pydantic；解析/类型失败 → 整批 DLQ + 返回 `[]`（绝不漏半条
  未校验记录进聚合）。
- **C6**：LLM 调用写 `llm_call_logs`（prompt_version/model/tokens/latency/success/schema_valid）；
  prompt 模板 `quark_extract_v1.md` 版本化永不覆盖。
- **C7**：LLM 超时/网络/JSON 垃圾 → DLQ + `[]`，绝不 raise（提取是富化，不能阻断捕获）。

镜像 `realityos_insights._base` 的 `_llm`/`_log_call`/`_safe_dlq` rails，两个 insight 生产者
共享同一失败纪律。

## 不做（诚实边界）

- **不**做 Time/Behavior/Context/Network 4 类 Quark（防空跑，依赖被砍链路）。
- **不**做 aux R1 SelfState 聚合（需声学 intensity，Phase 2.5）。
- **不**把 Meaning Quark 提升为 R2 Task（不编造任务生命周期）。
- **不**自动触发（B3 才接线调度/hook；本批 `register` no-op，显式调用）。

## 验证

- `tests/plugins/test_adr_v6_049_quark.py`（11）：Protocol runtime_checkable、extract 三类
  happy、C5 invalid JSON→[]+DLQ+log、C7 LLM 异常→[]+DLQ+log、later-phase kind 被忽略、
  空捕获短路不调 LLM、聚合 PRIMARY 落点（identity/meaning R7/feeling R9）、confidence 传递、
  无 aux R1 伪造、闭环 extract_and_aggregate、失败诚实返回。
- ruff 净；CI 三连随提交核验。
