# ADR-V6-043：G1 RAG 硬引用门（F2）— 可信度根基

- **状态**：Accepted（2026-07-20）
- **日期**：2026-07-20
- **Phase**：2-A（消费者层 G1 / P0 反假绿）
- **关联**：ADR-V6-040（母纲 / P0 清单 F2）、strategy-02（G1 可信度）、danao13 `rag_service.py:711-731`（生产引用范式）、ADR-V6-041/F1（.pyc 防御）、ADR-V6-042/F6（样本门）
- **铁律**：C1（一 ADR + 测试）、C5（agent 答案须通过 schema/接地校验）、C7（无静默失败 — 不接地断言是可信度静默失败）

---

## 背景

strategy-02 与 Agent ② 诊断：V6 的整个可信度体系建立在一个未经校验的假设上——**agent 关于用户过去的答案是否真的扎根于召回的 memo**。F2 核验为真：

- `provider.prefetch` 把召回渲染为 `- {text[:200]}` 纯 bullet，**无 memo_id、无索引句柄**——agent 拿到了上下文，却没有任何方式标注某条陈述来自哪个片段。
- `ptg_search` 工具返回裸 `results` 行，同样无引用句柄。
- 没有任何代码校验答案里的断言是否被真实召回片段支撑。一条无依据的断言（"你上周说想辞职"，而根本不存在这样的 memo）与一条有依据的断言，对用户而言**完全无法区分**。这就是 G1 可信度根基失效。

danao13 V5 生产已有对症范式（`rag_service.py:711-731`）：合成 1-based 索引 `[idx]` → LLM 在 JSON 里返回 `citation_indices` → 边界检查 → 映射到真实 memo_id → C5 pydantic 门 + C7 DLQ 兜底。但 V5 是专用 `/chat` 端点（`response_format=json_object` 强制结构化输出）；**V6 是 agent 驱动（hermes-agent 主循环自由文本答案）**，无法直接套强制 JSON。

---

## 决策

### D1：建公共引用门 `plugins/memory/ptg/citation.py`（纯函数）

无 store/LLM 依赖，完全可单测。API（对齐 danao13 范式，适配 V6 自由文本）：
- `number_chunks(hits, *, max_chars=200) -> (chunk_text, index_map)`：渲染 `[idx] YYYY-MM-DD: <snippet>`（合成 1-based 索引 + 日期前缀），返回 `idx→memo_id` 映射。**真实 memo_id 永不进 prompt**——越界幻觉索引在 `validate_citations` 边界检查时被丢弃，不会泄露真实 id。
- `extract_cited_indices(text) -> list[int]`：从答案文本解析 `[N]` / `[N,M]` / `[N-M]` 引用 token（支持 ASCII/全角逗号、连字符、en-dash；范围展开上限 50 防 `[1-9999]` 病态）。
- `validate_citations(cited, index_map, hits) -> (sources, dropped)`：边界检查（对齐 danao13:607-609,724-731）。越界/幻觉索引 → `dropped`（遥测，高 dropped=LLM 在编造引用）；有效索引 → 映射 `{"memo_id","date","snippet","index"}`（去重）。
- `ground_answer(answer, hits) -> dict`：端到端（number→extract→validate）→ `{"sources","dropped","cited_indices","has_valid_citation","n_chunks"}`。
- `looks_like_history_claim(answer, terms) -> bool`：启发式——答案是否做出了**关于用户过去、本应被接地**的断言（含召回词 或 过去时/近义词标记）。**故意宽松**（false positive 可恢复，false negative 让门失效），故偏向 True。

`CITATION_INSTRUCTION` 常量：进 `system_prompt_block`，明确告诉 agent「引用用户过去时必须用 [N] 标注，不得引用未召回编号，无依据时如实说『我没有这方面的记录』」。

### D2：接线进 PTGProvider（四处）

1. **`system_prompt_block`**（memo 数>0 时）：追加 `CITATION_INSTRUCTION`——agent 被告知要引用。
2. **`prefetch`**：召回渲染为 `number_chunks` 的编号块（替代旧 `- {text[:200]}`），并把 `hits + query` 暂存到实例槽 `_last_recall_hits/_last_query` 供 `sync_turn` 校验。
3. **`_handle_search`**（ptg_search 工具）：返回 `numbered`（编号块）+ `citation_rule` + 原始 `results`；同样暂存 hits/query。
4. **`sync_turn`**：答案落地后调 `_observe_citation_quality(assistant_content)`——观察 + 记录，**不阻断**（C7）。

### D3：观察语义（诚实边界）

`_observe_citation_quality` 三态，全 observation-only：
- **grounded**：答案引用了 ≥1 有效召回片段 → bump `citation_grounded_turns`（健康信号）。
- **ungrounded**：答案含历史性断言（`looks_like_history_claim=True`）但零有效引用 → **WARN 日志 + bump `citation_ungrounded_turns`**。这是 G1 可信度事件。计数器持久化进 `ptg_meta`，跨重启可查可信度漂移。
- **neutral**：本轮无召回在作用域，或答案是无历史断言的泛回复 → 不计。

**硬强制（refuse-to-render）需要 agent 主循环答案钩子**——这是 hermes-agent 核心改动，超出本 ADR 范围。本 ADR 交付的是**可观察的地基**：编号句柄 + 校验器 + 计数器。下一迭代（文档化为 ADR-V6-0xx）在 agent 主循环挂答案后处理钩子，把 ungrounded 历史断言在渲染前拦截/降级。**不假装已硬强制**——这是诚实的边界声明，非假绿。

### D4：测试（C4）

- `tests/test_citation_gate.py`（单测）：number_chunks 编号/日期/空集；extract 解析单/多/范围/全角/越界；validate 边界/去重/dropped；ground_answer 端到端；looks_like_history_claim 召回词/过去时/泛回复。
- `tests/plugins/memory/ptg/test_citation_wiring.py`（接线集成）：prefetch 渲染编号块 + 暂存 hits；_handle_search 返回 numbered；sync_turn grounded→计数器+1；sync_turn ungrounded 历史断言→ungrounded 计数器+1；sync_turn 泛回复→不计；空召回→不校验；store 故障→fail-open。

## 后果

- G1 可信度根基补上：agent 现在有引用句柄（编号块）+ 被告知引用（指令）+ 答案被校验（ground_answer）+ 可信度事件被计数（ptg_meta）。
- ungrounded 历史断言不再静默——WARN 日志 + 计数器，可信度漂移可观测。
- 下一迭代硬强制（refuse-to-render）的契约已就位：校验器 + 计数器，只需挂 agent 主循环答案钩子。
- 与 F1（.pyc 防御）、F6（样本门）共同构成 P0 反假绿三件套：根因（.pyc 单点）→ 数据信任（样本门）→ 答案信任（引用门）。
