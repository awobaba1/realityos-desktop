# ADR-V6-002: 操作电脑能力继承范围（REV-7 决策①）

- **Status**: Accepted
- **Date**: 2026-07-17
- **Refs**: V6 架构设计 §0.6 决策① / §2.2 / Phase 0 实施计划

## Context
用户要求「保留 hermes + openclaw 所有操作电脑能力」（§0.6 决策①）。核实：hermes `tools/computer_use/`（CUA 后端 + 视觉路由 + 权限层）/ 终端 / 文件 / 代码执行 / 浏览器×4 / Mac 原生（`skills/apple`）/ 智能家居 / 168 技能 / MCP / 委派 / SWE——**全谱系原生**；openclaw **无原生 computer-use 模块**，其价值在五处增强子系统。

## Decision
- **hermes 操作电脑能力 = INHERIT 全量**（fork 即得，零新建）：`computer_use` / `terminal` / `file_operations` / `code_execution` / `browser×4` / Mac / `skills` / MCP / `delegate` / SWE。
- **openclaw 五处增强 = BORROW 设计，Python 重实现**（net-policy / ClawHub 三件套 / commitments / ACP / daemon），**不搬 TS 代码**（保 §2.1 单语言决策 + 2 人团队可行性）。

## Consequences
- ✅ 操作电脑能力 = 白送，Phase 1 先原样跑，不分散新建精力。
- ✅ 单语言栈（Python）保住，避免 TS+Python 双语言摩擦。
- ⚠️ openclaw 五处重实现有工作量：net-policy Phase 0 固化（§6.5），其余排消费层主线后。
- ⚠️ computer-use 读屏的数据边界 → 见 ADR-V6-003。
