# ADR-V6-072 — PyPI Trusted Publishing 连红：用户侧注册操作（诚实 defer）

**状态**: Accepted（诚实 defer —— 非代码可修，用户侧 pypi.org 操作）
**日期**: 2026-07-21
**关联**: 第四轮全量反假绿审计（C1/CI 维度）/ [ADR-V6-037](ADR-V6-037.md)（release.py autopublish）/ [ADR-V6-084](ADR-V6-084.md)（desktop-build）/ [ADR-403](../../adr/ADR-403.md)（密钥不入库）/ [ADR-V6-038](ADR-V6-038.md)（诚实化方法论）

## 背景（第四轮审计：PyPI workflow 连红未披露）

第四轮审计 C1/CI 维度发现 `.github/workflows/upload_to_pypi.yml` 自启用 OIDC Trusted Publishing 起连续失败（≥4 次），且 Release notes 未披露。亲核最新失败 run（29764432402）日志：

```
##[error]Trusted publishing exchange failure:
  invalid-publisher: valid token, but no corresponding publisher
  (Publisher with matching claims was not found)
```

即：workflow 产出的 OIDC token 合法，但 **pypi.org 上未注册匹配的 Trusted Publisher** ——「做了 workflow，没配 publisher」的 ADR-037 同型（producer 在，consumer 侧缺）。

## 决策：诚实 defer（非代码可修）

这是**用户侧操作**，代码层无法修复 —— workflow 配置本身正确（OIDC `id-token: write` + `environment: pypi` + `pypa/gh-action-pypi-publish`，无 API token，符合 ADR-403 密钥不入库）。唯一缺口在 pypi.org 控制台。

### D1：用户侧注册步骤（一次配置）

在 https://pypi.org/manage/project/hermes-agent/settings/publishing/ 添加 GitHub Trusted Publisher，填入：

| 字段 | 值 |
|---|---|
| PyPI Project | `hermes-agent` |
| GitHub Owner | `awobaba1` |
| GitHub Repository | `realityos-desktop` |
| Workflow filename | `upload_to_pypi.yml` |
| Environment name | `pypi` |

注册后下一次 `v20*` tag push（或 Actions → upload_to_pypi → Run workflow 手动触发）即发布成功。

### D2：分发不被阻断（PyPI 是次要路径）

PyPI 红不阻断桌面分发：`scripts/release.py` 推 tag → `desktop-build.yml` 构建 mac dmg + win exe 并挂到 GitHub Release（**主分发通道，已验证 v2026.7.22 资产真实**）。`pip install hermes-agent` 是次要通道（CLI-only wheel），当前断 —— 待用户完成 D1 注册后恢复。

### D3：诚实化（反假绿）

- 本 ADR 显式记录「PyPI 连红 + 未注册 publisher」—— 不藏。
- v2026.7.22 Release notes 须披露：PyPI 通道当前断、恢复需用户侧注册。
- 静态守卫（`tests/ci/test_adr_v6_072_pypi_trusted_publishing.py`）：断言 workflow 保持 OIDC trusted publishing（`id-token: write` + 无 `password`/`API_TOKEN` 注入）。防止有人为「修绿」塞 `PYPI_API_TOKEN` secret —— 那会把长期密钥引入 CI（违 ADR-403）。断红正确，断绿（改塞 token）才是真倒退。

## 反假绿自检

- 非代码可修 → 显式记 defer + 用户操作步骤（不假装「已修」）。
- 分发主通道（dmg/exe）真实可用（v2026.7.22 资产已核）。
- 守卫防错误「修复」（塞 token）而非防正确修复（注册 publisher）。

## 后续

- 用户完成 D1 后：手动触发一次 upload_to_pypi 验证 `hermes-agent` 上 PyPI；成功后从本 ADR 状态改为「已闭环」并更新 Release notes。
