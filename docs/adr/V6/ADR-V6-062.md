# ADR-V6-062 — ADR-V6-025 激活咽喉 C4 端到端守卫（抽取守卫为可测纯函数）

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-053](ADR-V6-053.md) P2-T3 / [ADR-V6-025](ADR-V6-025.md) / 铁律 C4（每个缺陷成为测试用例） / [ADR-V6-037](ADR-V6-037.md) 反假绿

## 背景（审计发现激活咽喉零端到端测试）

ADR-V6-025 引入数据主权 dormant 守卫：阻止外部 SaaS 内存后端（honcho/hindsight/mem0/retaindb/supermemory）被激活为 live `memory.provider`——一旦激活，`sync_turn` 会把用户对话（PIPL L3）发往第三方云主机，违反 V6「零服务器/数据主权」招牌承诺。

但 ADR-V6-053 P2-T3 复核发现：既有测试（`test_realityos_security_wiring.py::TestExternalMemoryDormantGuard`，5 例）**只钉私有谓词 `_v6_should_dormant`**。ADR 故事的核心——「`agent_init` 拒绝 `add_provider` + WARNING + 回退本地」——**零端到端测试**。

**C4 缺口的实证危害**：守卫谓词与激活接线是两层。若有人把 `agent_init.py:1455` 的 `if _v6_should_dormant(...)` 改成 `if False`（或删 WARNING，或调换 if/else 分支），谓词测试全绿，但激活守卫已被静默绕过——用户对话静默外泄。这是「测试覆盖假象」型假绿：CI 绿，但被测的不是真正的安全边界。

根因：守卫激活逻辑（load → available? → dormant? → warn+null → else add_provider）内嵌在 `agent_init` 的超长初始化函数（最近 `def` 在 867 行）里，**不可单元测试**——驱动该函数要 mock 整个 agent 初始化链（config/session_db/platform/...），不现实。不可测 = C4 缺口。

## 决策

### D1：抽取守卫激活逻辑为可测纯函数（等价重构）

把 `agent_init.py` 的内联守卫块抽取到 `plugins/memory/__init__.py::_activate_memory_provider_guarded(manager, name, load_fn) -> bool`：

```python
def _activate_memory_provider_guarded(manager, name: str, load_fn) -> bool:
    mp = load_fn(name)
    if not (mp and mp.is_available()):
        return False
    if _v6_should_dormant(name):
        logger.warning("realityos sovereignty: ...", name)
        return False
    manager.add_provider(mp)
    return True
```

`agent_init` 改为一行调用：
```python
from plugins.memory import _activate_memory_provider_guarded
_activate_memory_provider_guarded(agent._memory_manager, _mem_provider_name, _load_mem)
```

**为何放 `plugins/memory`**：守卫谓词 `_v6_should_dormant` 已在此模块；激活是谓词的消费方，归一到数据主权模块更内聚。分层合规：`plugins/memory`（lower）被 `agent_init`（upper）调用，不反向。

**为何注入 `load_fn`**：测试可传 fake loader，无需 monkeypatch 模块全局。`manager` 同理（测试传 fake manager 带 `add_provider`）。函数对 `manager` 的唯一依赖是 `add_provider(mp)`——不访问 `manager.providers`（那个判断留在 `agent_init:1466`）。

### D2：等价性证明（行为不变）

抽取前后对照 `agent_init:1444-1465` + `1466`：

| 输入 | 抽取前 | 抽取后 | 等价 |
| --- | --- | --- | --- |
| load 成功 + available + 非 dormant | add_provider → providers 非空 → 1466 True → initialize_all | 函数返回 True（已 add_provider）→ providers 非空 → 1466 True | ✅ |
| dormant（外部 SaaS） | warn + `_mp=None` → 不 add → providers 空 → 1466 False → else debug + manager=None | 函数 warn + 返回 False → 不 add → providers 空 → 1466 False → else | ✅ |
| load 返回 None / unavailable | `_mp` falsy → 跳过 if → providers 空 → 1466 False → else | 函数返回 False → providers 空 → 1466 False → else | ✅ |

`_mp` 变量在守卫段（1466 行起）之后不再被引用（1466 用 `manager.providers`，1512 用 `_mem_provider_name`）——抽取零后续依赖风险。`_load_mem` 从「直接调用」改为「作为参数注入函数内调用」，调用时机与参数不变。

### D3：sovereignty warning logger 归一化

抽取前 WARNING 落在 `agent_init` 的 logger；抽取后落在 `plugins.memory` logger。**消息内容与触发条件完全不变**。归一理由：sovereignty 是 memory 模块的职责，warning 应与守卫同处。既有测试 `test_discovery_does_not_log_sovereignty_warning`（断言 discovery 路径不 log sovereignty）只覆盖 `load_memory_provider`，不覆盖 activation——故 logger 迁移零回归（已本地实证：9 例谓词测试 passed）。本 ADR 显式记录此 logger 变更以备追溯。

### D4：端到端测试钉激活咽喉契约

`tests/plugins/memory/test_adr_v6_062_sovereignty_activation_guard.py`（C4）：

- **`TestActivateMemoryProviderGuarded`**（6 例）：
  - `test_activates_local_provider`：holographic（本地）→ add_provider 被调用 + 返回 True
  - **`test_refuses_every_external_saas`**（参数化遍历 blocklist，**承重**）：每个外部 SaaS → add_provider **未调用** + sovereignty WARNING + 返回 False。直接钉死「dormant name 永不达 add_provider」契约——C4 缺口关闭。
  - `test_escape_hatch_allows_external_activation`：env=1 + honcho → add_provider 被调用
  - `test_unavailable_provider_not_activated`：is_available()=False → 不 add
  - `test_not_found_returns_false`：load 返回 None → 不 add
  - `test_dormant_warning_on_plugins_memory_logger`：D3 的 logger 归一化钉死（warning 在 plugins.memory logger + 含 provider 名）

- **`TestBlocklistEmpiricalInvariant`**（实证不变量，反「凭名字」假绿）：
  - `test_every_blocklist_entry_has_cloud_egress_signal`（参数化遍历 blocklist）：每项插件源码必须含第三方云 egress 信号（非 loopback https URL / cloud SDK MemoryClient / API_KEY env）。钉死 ADR-025「blocklist 凭实证非凭名字」——防未来误加本地插件或漏加 cloud 插件。
  - `test_local_holographic_has_no_cloud_egress`：holographic（本地 SQLite，白名单对照）源码**无**第三方云 URL。实证确认本地插件不该被 block。

## 不做（诚实边界）

- **不**重构 `agent_init` 整个超长函数——只抽取守卫这一小段（最小手术面，降低回归面）。`agent_init` 的其他不可测段落是独立的测试债务，超出本 ADR。
- **不**改 `load_memory_provider`（discovery 路径）——守卫仍在 activation 非 discovery（ADR-025 D1 不变）。
- **不**改 blocklist 成员——本 ADR 只补测试钉死既有 blocklist 的实证性 + 激活咽喉。blocklist 成员变更须另起 ADR。
- **不**声称守卫防「所有」数据外泄——它只防 memory.provider activation 路径；LLM egress（§6.6）、工具 fetch（url_safety）是独立守卫面，不在本 ADR。

## 验证

- 抽取后 import 实证：`_activate_memory_provider_guarded` 可 import + `agent_init` 可 import（零语法/导入破坏）。
- 等价回归：既有 `TestExternalMemoryDormantGuard`（9 例谓词测试）passed——logger 迁移零回归。
- 全 memory provider 测试套回归（`tests/agent/test_memory_*.py` 8 文件用 add_provider）——本地实证零回归后提交。
- 新测试 `test_adr_v6_062_sovereignty_activation_guard.py`（8 例）本地绿。
- `ruff check .`（全量）净。
- 提交后 CI 三连核验。

| 抽取是否真等价（非行为变更） | ✅ | D2 三分支对照表 + `_mp` 后续无引用实证 + 9 例谓词测试零回归 |
