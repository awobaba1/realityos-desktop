# ADR-V6-065 — DLQ 读面 + resolved 状态激活（C7 Phase-Gate backlog 可观测）

**状态**: Accepted
**日期**: 2026-07-21
**关联**: [ADR-V6-053](ADR-V6-053.md) 第三轮审计 R2 / [ADR-V6-063](ADR-V6-063.md)（同类读面：citation）/ [ADR-V6-037](ADR-V6-037.md)「做了没发」/ 铁律 C7 / Phase-Gate Checklist「DLQ backlog < 5/week」

## 背景（第三轮审计发现 DLQ 写了无读面）

ADR-V6-053 第三轮审计 R2（消费层闭环）发现：`dlq_messages` 表有 5+ 活跃 producer（atomize/quark/theory/insights/provider，C7 每条失败路径都 `insert_dlq`），但**零消费者**：

- 全库 `SELECT ... FROM dlq_messages` 仅命中 `tests/`；无 `hermes dlq` 子命令、无 `/api/dlq`、无 UI。
- `resolved` 列（`schema.py:441`）+ `idx_dlq_resolved` 索引（`schema.py:444`）**设计预期翻状态，但全库无任何 UPDATE 把 resolved 翻 1** —— 死字段。

这是 ADR-V6-037「做了没发」最致命假绿类的又一例：**C7 Phase-Gate Checklist 明列「DLQ backlog < 5/week」为门禁指标，但代码无任何读取路径，指标不可观测，所有 DLQ 行写了就沉**。比 ADR-063（citation 计数器写了无读面）更严重 —— citation 是观测，DLQ 是 C7 门禁本身。

## 决策

### D1：PTGStore.dlq_stats（聚合，Phase-Gate backlog 可观测）

`dlq_stats(user_id=None) -> {total, unresolved, resolved, by_source, by_error_type}`。读聚合，never raises（C7）。这是 Phase-Gate「DLQ backlog < 5/week」KR 的读面。

### D2：PTGStore.dlq_list（明细）

`dlq_list(user_id=None, resolved=None, source=None, limit=50) -> List[Dict]`。newest-first，过滤 user/resolved/source，limit 夹 `[0, 200]`。读明细，never raises。

### D3：PTGStore.dlq_resolve / dlq_resolve_all（激活 resolved 死字段）

`dlq_resolve(dlq_id, user_id=None) -> bool` + `dlq_resolve_all(user_id=None, source=None) -> int`。把 unresolved 翻 `resolved=1` + `resolved_at`。

**append-only 语义合规**：`resolved`/`resolved_at` 是**状态元数据**，非 event 内容。UPDATE 符合 append-only 精神（append-only 保护失败 payload 不可改；resolution 是新增状态层）。schema 的 `idx_dlq_resolved` 索引 + `retry_count`（同性质 DEFAULT 0 递增）即为设计预期。失败 payload（`error_msg`/`original_data`）永不改。

### D4：hermes dlq CLI（薄委托，类比 citation/k_cmd）

`hermes dlq`（默认 stats + 最近未解决 list）／`--stats`／`--resolved`／`--source <s>`／`--limit N`／`--resolve <id>`／`--json`。诚实空状态「尚无 DLQ 记录」+ backlog soft hint（≥5 unresolved ⚠️，非 gate）。

### D5：main.py 接线 + _BUILTIN_SUBCOMMANDS

4 点接线（import `build_dlq_parser` + `_BUILTIN_SUBCOMMANDS` 加 `"dlq"` + `cmd_dlq` thin delegate + `build_dlq_parser` 注册），同 ADR-063 citation 模式。

### D6：测试钉死（C4）

`tests/plugins/memory/test_adr_v6_065_dlq_read_surface.py`：store 层（dlq_stats 空/读写/聚合、dlq_list 过滤/limit、dlq_resolve 翻状态/idempotent、dlq_resolve_all 计数）+ CLI 渲染（空状态/stats/list/resolve/json）。

## 不做（诚实边界）

- **不**做自动 alert（告警通道是另一层基础设施，本 ADR 只补读面）。
- **不**删 DLQ 行（append-only 永不删；resolved 翻状态≠删除）。
- **不**改失败 payload（`error_msg`/`original_data` 永不改；只翻 resolved 状态元数据）。
- **不**把 backlog hint 做成硬 gate（观测非阻断，与 citation soft threshold 同口径）。

## 反假绿（本 ADR 自身的诚实性）

| 项 | 判定 | 依据 |
