# ADR-V6-042：sample-size 分级 confidence 门（F6）— V6 最大假绿源修复

- **状态**：Accepted（2026-07-20）
- **日期**：2026-07-20
- **Phase**：2-A 前置（P0 反假绿）
- **关联**：ADR-V6-040（母纲 / P0 清单 F6）、ADR-V6-017（weekly mirror）、ADR-V6-018（daily report）、strategy-02、danao14 `realityos_correlation/compute.py:23-27`
- **铁律**：C1（本决策一 ADR + 测试）、C5（confidence 门控）、C7（无静默失败 — 假强结论是静默失败）

## 背景

Agent ③ 诊断（strategy-02 印证）：V6 `insight_aggregation.confidence` 由**粗粒度冷启动指标**（memo 总数 + 注册天数，经 `InsightReportService.CONFIDENCE_MAP`）定级。一周 50 条 memo 但只有 2 条 R9 情绪原子 → 情绪结论基于 n=2,但 confidence 仍 0.8「sufficient」。

**这是 V6 单一最大假绿源**：LLM 乐意在 2 个数据点上写具体强烈的情绪叙事,缓存把它当高可信洞察呈现给用户。danao14 `_grade_confidence`（`compute.py:23-27`,PRD §8.6.7）是对症药 —— 纯计数器硬门,小样本永不产出强结论 —— 但它只在 K域单点用,未泛化到所有 insight 输出。

---

## 决策

### D1：提取 sample-size 分级为公共门 `plugins/memory/ptg/confidence_gate.py`

纯函数模块（无 store/LLM 依赖,完全可单测）。PRD §8.6.7 阈值（不可改,改需 ADR）：
- `< 10` → `CONF_FLOOR` (0.1)（too few,信号"勿信"）
- `10-29` → `CONF_MID` (0.6)
- `≥ 30` → `CONF_HIGH` (0.9)

API：
- `grade_confidence_by_sample(n) -> float`：单样本分级（None 不可持久因列 CHECK 0..1,故 floor）
- `sample_sufficiency(n) -> str`：映射 insufficient/partial/sufficient
- `cap_confidence_by_atom_samples(base, counts, kinds, *, skip_absent=True) -> (capped, label, kind)`：**接线点** —— cap = min(base, 最弱**在场**kind 样本级)。缺席 kind 跳过（不驱动结论,prompt 省略空 section）;全缺席 → 防御性 floor
- `merge_sufficiency(*labels)`：多 verdict 取最悲观
- `is_valid_sufficiency_label(label)`

### D2：接线进 `InsightReportService.generate()`（_base.py）

冷启动 `data_sufficiency` **保持不动**（它 gate 生成-vs-placeholder,非信任度）。在 confidence 计算后加 cap：
```python
sample_conf, sample_suff, sample_kind = cap_confidence_by_atom_samples(
    confidence, agg["atom_counts"], self.SAMPLE_GATE_KINDS)
confidence = sample_conf
agg["sample_sufficiency"] = sample_suff   # 可追溯
agg["sample_weakest_kind"] = sample_kind
```

`SAMPLE_GATE_KINDS` 默认 8 类原子（R3_Person/R2_Task/R7/R8/R12/R1/R9/R0）,子类可收窄。

**语义**：mirror 仍生成（冷启动过门）,但缓存 confidence 诚实反映"生成内容多可信"。n=2 情绪叙事 → 缓存 0.1 + sample_sufficiency=insufficient,绝不缓存为 0.8 高可信。

### D3：测试（27 项）

- 23 单测（`test_confidence_gate.py`）：阈值边界 / skip_absent 双模式 / merge / 全缺席 floor
- 4 集成测（`test_sample_confidence_gate_integration.py` 含 2 核心反假绿）：
  - **thin sample 端到端**：40 memo + 30 天注册（冷启动 sufficient,LLM mirror 生成）+ 仅 2 R9 原子 → 断言 cached confidence == CONF_FLOOR + sample_sufficiency=insufficient + weakest_kind=R9_Emotion
  - **abundant control**：每类 ≥30 → cap 不误伤,confidence 保持 0.8 + sample_sufficiency=sufficient

## 反假绿：本 ADR 的诚实性

| 项 | 判定 | 依据 |
| --- | --- | --- |
| F6 缺口 | 核验为真 | _base.py CONFIDENCE_MAP 仅冷启动,无样本门 |
| 端到端真触发 | 集成测实证 | thin sample 实跑 cached confidence floor 到 0.1 |

## 后果

- V6 最大假绿源（F6）消除：LLM 不再能在 2 数据点写强结论并被缓存为高可信
- 公共门已接到 K域 A2（`k_correlation.py:163`）与 InsightReportService（`_base.py:133`）；quark B1（`aggregation.py`）与 theory B2（`engine.py`）**暂未接 F6**（其单 record / 单 derivation 置信度语义不天然适配 report-level cap，若未来需统一须另起 ADR）—— 接线范围措辞纠错见 [ADR-V6-061](ADR-V6-061.md)
- 冷启动 gate 与样本 gate 职责分离：前者管"够不够生成",后者管"生成的多可信"
