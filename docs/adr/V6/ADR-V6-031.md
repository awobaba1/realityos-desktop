## 背景

ADR-V6-022 审计（镜头 10）暴露一条「承诺膨胀」（桶 C）：

- **action 14**：架构总纲 §4.3C（母纲笔误为 §8 line 343-349）把 `AcousticProvider`（子类化 `TranscriptionProvider`、覆写 `transcribe` 出 SER/韵律/分离富字段）列为 **Phase 1b** 承诺，但 :349 自己又写「Phase 1 仅 STT；Phase 2.5 SER+分离」——叙事自相矛盾。
- **action 15**：STT 首启模型预下载未实现——faster-whisper `base`（~150MB）在**首次 transcribe** 时才懒下载，首启体验差且无进度提示。

母纲决策：action 14「删 §4.3C Phase 1b 富字段承诺，移 Phase 2.5；`asr_quality_factor` 接口保留；**不做** B 路径（现在造 AcousticProvider）——超 Phase 1b 范围」；action 15「installer 预拉 faster-whisper base 或 settings 加下载进度」。

## 亲核调查结论（反假绿）

调查（Explore 41 tool-use）发现一个**比预想更乐观**的事实：

1. **fork 源码本就诚实**：全仓 `grep AcousticProvider` 命中为零——**类根本不存在**。这不是「写了没接线的孤儿」（如桶 B 的 sovereignty/security），而是「从未写过、只在文档里'覆写'了」。fork 当前 STT 走 Hermes 内建 `tools/transcription_tools.py` 的 `local` provider（faster-whisper），注册表 `agent/transcription_registry.py:27-31` 拒同名插件，AcousticProvider 永不被消费。**代码侧无需任何降级动作——它已经在正确的 Phase 1 状态**（仅 `TranscriptionProvider` ABC 默认 `transcribe()`）。
2. **fork 文档本就诚实**：fork 内 `AcousticProvider` 唯一命中是 `docs/adr/V6/ADR-V6-022.md`（审计诊断本身，非承诺）。§4.3C 过度承诺的叙事在 **V5 仓库** `/Users/wugang/danao13/RealityOS-V6-架构设计.md:343-349`（架构总纲，非 fork 分发物）。故 **fork 分发物不携带该虚假承诺**。
3. **`asr_quality_factor` 接口已就位**：`plugins/memory/ptg/confidence.py`（默认 1.0）+ ADR-V6-016:55/60/143 钉死。Phase 2.5 建 AcousticProvider 时直接填，零迁移。
4. **STT 路径澄清**：fork 是「Hermes fork + 4 个 realityos_* 插件（calibration/insights/security/sovereignty）」，**无 voice/STT 插件**。故 fork 的 STT = Hermes 内建 faster-whisper（transcription_tools.py），`DEFAULT_LOCAL_MODEL = "base"`（transcription_tools.py:88），懒下载于首次 transcribe（:1121-1146）。action 15 对 fork 分发**有效**（改善 Hermes 转写工具首启 UX）。注：RealityOS V5 的云 ASR（FunASR→GLM）是**另一条 V5 路径**，不在此 fork，本 ADR 不触碰。

## 决策

### D1：AcousticProvider 正式降级 Phase 2.5（fork 侧零代码改动）

fork 代码与文档**已在正确的 Phase 1 诚实状态**（无类、无虚假承诺）。本 ADR 的作用是**把母纲 action 14 的降级决策正式记录**（C1），并明确：

- **Phase 1（当前）**：仅 `agent/transcription_provider.py::TranscriptionProvider` ABC + 默认 `transcribe()`（envelope success/transcript/provider/error，无 SER/韵律/分离富字段）。**这是正确的**。
- **Phase 2.5**：建 `AcousticProvider`（子类化，覆写 `transcribe` 出富字段：STT→SER(HuBERT)→R9 / 韵律(openSMILE)→R1 / 分离(Pyannote)+声纹(ECAP)→R3 / 生物标记(wav2vec2)→R1）。届时填 `asr_quality_factor`。
- **`asr_quality_factor` 接口保留**：`confidence.py` 默认 1.0，ADR-V6-016 钉死，Phase 2.5 填。**不动**。
- **不做 B 路径**：现在造 AcousticProvider 超 Phase 1b 范围（母纲 action 14 明确）。

### D2：STT 首启模型预下载——**opt-in** best-effort 预拉（调查纠正母纲字面建议）

母纲 action 15 字面建议「installer 预拉 faster-whisper base」。**亲核调查推翻默认预拉**：

- `stt.faster_whisper` extra（`lazy_deps.py:132`）拉 faster-whisper==1.2.1 + ctranslate2 + tokenizers + onnxruntime + huggingface_hub 等，**~500MB+ 包**，再加 150MB 模型。默认 install 强制预拉会**破坏刻意的懒加载设计**（`_try_lazy_install_stt` / `lazy_deps.ensure` 就是为「按需装」而存在）、膨胀**每次**安装（含从不用本地 STT 的用户）。
- 现有首启路径**已诚实**：`transcription_tools.py:1146` 首次加载即 `logger.info("Loading faster-whisper model ... first load downloads the model")`——非静默，用户可见。

故正确工程判断：**prefetch 改 opt-in**（非默认 run-sequence），为「离线优先 / 已知要用本地 STT」的用户提供预拉能力，不强制全民下载重依赖。这是反假绿——调查发现母纲字面 call 是错的，做正确事并记录。

实现：
- **新增 `scripts/prefetch_stt_model.py`**：纯逻辑入口，`ensure("stt.faster_whisper", prompt=False)` 装包 → `WhisperModel("base", device="cpu", compute_type="int8")` 触发 HF 下载。**fail-open（C7）**：任何失败 `exit 0` + stderr 显式说明「预拉失败，首次转写懒下载 ~150MB」，**非静默**——lazy-load 兜底。
- **install.sh / install.ps1 加 `stt-model` stage case**（`run_stage_body` 分支 + `stage_needs_user_input` 返回 false），**但不进默认 manifest run-sequence**——仅 `hermes install --stage stt-model` 显式调用。opt-in。
- **不动默认安装体积/时长**：未显式请求 `stt-model` 的安装行为零变化。

## 验证

- **D1**：`grep -rn "class Acoustic\|realityos_acoustic" --include="*.py"` 源码侧 = 0（直到 Phase 2.5）；fork `.md` 内 AcousticProvider 命中 = 仅 ADR-V6-022（诊断）+ 本 ADR（降级记录），无 Phase 1b 承诺。`asr_quality_factor` 在 `confidence.py` 默认 1.0 不变。
- **D2**：install.sh/ps1 manifest 含 `stt-model` 阶段；`run_stage_body stt-model` 调 prefetch helper；helper 失败时 log_warning + return 0（fail-open）；单元测试 mock `WhisperModel` 构造验证 best-effort 契约。
- **三连核验**：headSha == CI `gh run view --json conclusion` 绿 + 内容指纹。

## 后果

- action 14 闭：AcousticProvider 降级 Phase 2.5 正式记录；fork 代码/文档早已诚实，零回归风险。
- action 15 闭：install 时 best-effort 预拉 faster-whisper base，首启转写不再有惊喜 150MB 等待（预拉失败亦有 lazy-load 兜底 + 显式 warning）。
- `asr_quality_factor` 接口为 Phase 2.5 预留（ADR-V6-016 不动）。
