# ADR-V6-036：桌面默认页（数据资产首页）选项（action 21）

- **状态**：Accepted（Phase 1b 收口·批次 3，2026-07-20）
- **日期**：2026-07-20
- **Phase**：1b（诚实化）
- **关联**：ADR-V6-022（母纲，桶 C / action 21）、ADR-V6-012（记忆读取回路/记忆浏览页）、ADR-V6-020（洞察读取 API/UI）
- **铁律**：C1（启动落点即决策，版本化 localStorage key）、C4（launch-view store 回归测试）、C6（i18n 4 locale 平价契约）

---

## 背景

ADR-V6-022 桶 C / action 21：「桌面默认页 = 数据资产首页选项」。

桌面冷启动**硬编码**落在 `NEW_CHAT_ROUTE = '/'`（新聊天）。`use-desktop-integrations.ts` 的冷启动 effect 仅做「恢复上次路由 / 上次会话」回退——用户**无法选择**「每次打开都先看记忆/洞察」这类数据资产首页。对一个以「捕获的原子」为核心资产（ADR-V6-012 记忆浏览页、ADR-V6-020 洞察页）的产品，缺一个「把数据资产设为首页」的开关是体验缺口。

## 决策

### D1：launch-view store（新，照 backdrop.ts + session.ts 模式）

新 `apps/desktop/src/store/launch-view.ts`：
- `LaunchView = 'chat' | 'memory' | 'insights'`，默认 `'chat'`。
- `getLaunchView(): LaunchView`——同步冷启动读（照 `getRememberedRoute`），`coerce` 健壮性（脏值→`'chat'`，永不崩冷启动）。
- `$launchView` atom（UI 绑定用）+ `setLaunchView(view)`（防非法值 no-op）。
- `launchViewRoute(view): null | string`——`'memory'→'/memory'`、`'insights'→'/insights'`、`'chat'→null`（**null = 不覆写，落回遗留恢复行为**）。路径与 `contrib/controller.tsx` 注册的贡献路由一致。
- 持久化 key `hermes.desktop.launchView.v1`（`.v1` 后缀 = 版本化，C1；命名照仓库 `hermes.desktop.<thing>[.v1]` 约定）。

### D2：冷启动兑现（use-desktop-integrations.ts）

在既有冷启动 effect（`restoredRef` 单次守门 + `locationPathname !== NEW_CHAT_ROUTE` 排除副窗）**最前**插入 launch-view 覆写：
- launch-view 覆写**优先级最高**——它是用户显式选择的「首页」，压过「恢复上次路由/会话」。
- 经 `isOverlayView(appViewForPath(target))` 守门（记忆/洞察非 overlay，可通过；防未来误配 overlay 为首页）。
- `'chat'`（默认）→ `launchViewRoute` 返回 null → **零行为变更**，遗留恢复链路逐字不变（向后兼容）。

### D3：Settings UI（appearance-settings.tsx）

加一个 3 选项 `SegmentedControl`（聊天/记忆/洞察）`ListRow`，置于语言行之后（启动落点是最显眼的「首屏」偏好）。照既有 `embedOptions`/`toolOptions` 的 `satisfies readonly { id; label }[]` 模式（TS 类型门禁）。`triggerHaptic('selection')` 触感与既有控件一致。

### D4：i18n 4 locale 平价（C6）

`Translations` 契约（`i18n/types.ts`）加 5 键（`launchViewTitle/Desc/Chat/Memory/Insights`），en/zh/zh-hant/ja 四 locale 全覆盖（TS 契约门禁——漏任一 locale 编译红）。

## 不做（诚实）

- **不新增 AppView 枚举/路由**（记忆/洞察已是 `ROUTES_AREA` 贡献路由，`appViewForPath` 返 `'extension'`，`syncWorkspaceIsPage` 视为页面——零路由层改动）。
- **不从 Electron 主进程注入路由**（渲染层 `HashRouter` + `use-desktop-integrations` 是唯一启动路由缝，保持单一缝）。
- **不删/不改 remembered-route 行为**（`'chat'` 默认零变更；仅当用户显式选记忆/洞察才覆写）。
- **不做「仅首次」语义**（launch-view 是「每次打开都先看 X」的持久偏好，非一次性——否则被 remembered-route 永远压过，设置形同虚设）。

## 后果

- ADR-V6-022 action 21 收口：桌面新增「启动时默认页面」设置（聊天/记忆/洞察），用户可把数据资产设为首页；默认聊天 = 零回归。
- 反假绿：设置真生效（冷启动 effect 兑现，非摆设）；C4 store 测试钉死持久化/路由映射/脏值健壮性；i18n 4 locale 平价受 TS 契约门禁。
- 未来加新首页候选（如图谱页）：扩 `LaunchView` 联合 + `launchViewRoute` 映射 + UI 选项 + i18n + 测试。
