# ADR-V6-013：Phase 1b-1 词汇表注入——LLM 上下文携带已知实体（移植 V5 ADR-049）

- **状态**：Accepted（Phase 1b 首特性，2026-07-19）
- **日期**：2026-07-19
- **Phase**：Phase 1b（图谱质量基础设施）
- **关联**：V5 [ADR-049](../../../danao13/docs/adr/ADR-049-LLM上下文注入已知实体词汇.md)（同源决策，本 ADR 把它移植到 V6 Atomizer）、ADR-V6-011（心脏/物化）、ADR-V6-012（⑦ 双路径证死——本特性是图谱质量，**非** ⑦ 精度杠杆）
- **铁律**：C1（本 ADR）、C6（system prompt v11 不动——词汇表是 user-prompt 变量上下文，非模板覆写）、C3（真实数据驱动——200 样本 dump 证 within-memo dedup 仅 3/200，cross-memo 54 重复才是价值所在）

## 背景

V6 Atomizer 的 `_format_user_prompt` 当前只注入「时间 + 地点 + 源文本」。两个真实生产缺口：

1. **ASR 鲁棒性**：RealityOS 是语音 memo 产品，用户口述 → ASR 转写 → 常见专有名词被同音误识别（V5 实例：用户已有「字节跳动」实体，ASR 输出「字体跳动」，LLM 无词汇表无法判断是误识别 → 抽出错的 R0_Entity，或漏抽）。LLM 若能看见用户已知实体，可自纠正同音/近音误识别。
2. **图谱连贯性**：200 样本实证——127 次 R3 人物抽取里仅 73 个 distinct，**54 次是跨 memo 的同名重复**。无词汇表时，同一「张三」在不同 memo 被抽成独立节点 + 可能用不同称呼（张三/老张/张总），图谱碎片化。注入词汇表让 LLM 用 canonical 名 + aliases 一致命名。

## 决策

把 V5 ADR-049 的「已知实体词汇表注入 user prompt」移植到 V6 Atomizer，**system prompt 不动**（C6）：

1. **`store.list_top_entities(user_id, limit=100)`**：按 `mention_count DESC` 取 top 100 活跃实体（排除 `deleted_at IS NOT NULL` + self 节点 `properties.is_self`），返回 `entity_name / entity_type / mention_count / aliases`（aliases 从 `properties.aliases` 读）。
2. **物化补存 R3 aliases**：`_materialize_graph` 处理 R3PersonAtom 时把 `atom.aliases` 并入 `properties.aliases`（现行只存 sentiment/interaction_type，丢 aliases——这是 ADR-V6-011 决策6 的遗漏，本 ADR 补）。
3. **`Atomizer._build_entity_vocab()`**：调 `list_top_entities`，按 `entity_type` 分 4 桶（person→人物 / task→任务 / topic→话题 / context→情境），每桶 top 20，每条 `canonical 名（别名…）`。**全表 <100 实体或全桶空 → 返回 None（首次用户无实体，省 token，行为同现状）**。
4. **`_format_user_prompt` 加可选 `entity_vocab` 形参**：非 None 时在源文本前插入自描述段——
   ```
   ## 已知实体词汇（用于：① 把用户输入里的近似/同义称呼归一到下列标准名；
     ② 修正明显的 ASR 同音误识别，如「字体跳动」→「字节跳动」。不强制只抽这些。）
   [人物] 张三（老张、张总）; 李四; 妈妈
   [情境] 厦门国贸; 字节跳动
   [话题] …
   ```
   段头自描述指令（在 user prompt 变量区），**不进 system prompt**——故 `PROMPT_VERSION` 仍是 v11（C6 合规：未覆写任何 prompt 模板）。

## 成本

- 额外 ~150-400 tokens/次（仅当用户已有实体时；首次用户省略）。DeepSeek-Flash 价 < ¥0.001/次。
- 每 turn 一次 `list_top_entities` 查询（top 100，已命中 `idx_entities_user_type` 索引）。

## 诚实边界（反假绿）

- **本特性不是 precision⑦ 的杠杆**。200 样本 dump 实证：within-memo 重复仅 3/200（5 原子），dedup/词汇表对 eval precision 影响**可忽略**。⑦ 的真因（prompt 忠实执行 V5 标准 vs 样本更窄口径）词汇表不治——见 ADR-V6-012 双路径证死。
- **eval 样本是净文本（非 ASR 输出）**，故 ASR 纠正收益在 eval 中**不可见**。本特性的价值在**真实生产**（用户口述 ASR 乱码鲁棒）+ **图谱连贯**（跨 memo 同名归一），将以**真实 ASR 冒烟 + 图谱去碎片度量**验证，**非** eval precision 数字。
- 若后续要给 LLM 更强的词汇表指令（超过 user-prompt 段头），则需 system prompt v13（C6 版本化）——本 ADR 暂不引入。

## 明确不做

- ❌ 全量实体注入（>1000）→ token 膨胀（同 V5 ADR-049 拒绝）。
- ❌ embedding 语义检索实体 → 需向量基础设施，Phase 2+。
- ❌ 跨实体共现边 / R1·R7 图节点（同 ADR-V6-011 留治）。
- ❌ system prompt v13（段头自描述已足；强指令留待实证不够时再加）。

## 核验计划

- `ruff check .` 全净；`pytest tests/plugins/memory tests/benchmark` 全量绿（+新增词汇表回归）。
- 真实 DeepSeek 冒烟：喂含 ASR 近音误识别的合成 memo（如「字体跳动」+ 词汇表含「字节跳动」），验证 LLM 用词汇表纠正。
- 图谱去碎片度量：同一人物多 memo 后 `entities` 表该人物为 1 行（mention_count 累加），非多行碎片。
- eval precision 测一次（诚实记录，预期 ⑦ 基本不变）。
