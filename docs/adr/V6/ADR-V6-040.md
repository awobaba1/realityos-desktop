## 决策

### D1：优先级重排 — 先 A（消费层闭环）后 B（合成层），ADR-V6-039 Batch1-3 推迟

**ADR-V6-039 不撤销**（IDL 契约 + 实现边界仍有效），但其 Batch1-3（quark+theory）**推迟到 A 层之后**。

**理由（Agent ④ 实证，非推断）**：
1. Phase 2 无声学下 theory 只能撑 **4/7 PC 维**（Time/Cognition/Emotion/Execution），且 **Cognition 因 V6 R8 无连续 score 严重降级**
2. Energy/Social 缺音频链路 = 不可；Environment 永不可（scene.py POC fail defer）
3. V6 fork 4 张 event 表与 danao14 列结构**完全不兼容**，每个 producer 都要适配层 → 非 drop-in 移植
4. **而 F2（G1 RAG 无引用校验）是整个可信度系统的根基** —— LLM 可幻觉引用，比「洞察广度不够」致命。strategy 02 判定为单点故障。

### D2：P0 反假绿清单（立即执行，全部已核验）

- **P0-1 反假绿基建**：F1（`.pyc 防御` conftest）+ atomize DLQ 兜底核验 + `deletion_log` 审计表
- **P0-2 sample-size 分级 confidence 门**：提取 `_grade_confidence`（<10 不发 / 10-30 中 / ≥30 高）为所有 LLM insight 公共前置门（F6 对症药）
- **P0-3 G1 RAG 硬强制引用**：对齐 danao13 生产 JSON 合成索引范式（`rag_service.py:711-731`），越界裁剪 + C5 schema gate + C7 DLQ + 无引用拒绝渲染（F2）
- **P0-4 store 层基建**：F3（atomizer R9 entity key）+ F4（transaction/upsert_relation delta/relations.stale_at/mark_k_correlation_stale）

### D3：Phase 2-A 消费层闭环（先 A，A1~A5）

G1 RAG（A1）→ K域 compute（A2，依赖 P0-4）→ R12 显式通路（A3）→ 纠错反馈修复（A4）→ M域人物画像（A5）。

### D4：Phase 2-B 合成层（后 B，ADR-V6-039 Batch1-3 推迟到此后）

quark（B1）→ theory LLM 近似（B2，**诚实降级**）→ 调度接线+UI（B3）。

**诚实降级铁律**：theory 输出必须标每 PC 维的 `basis` + `degraded`。**绝不**把 None 渲染成「平稳」或省略降级标 —— 那是假绿（Agent ④ 最高危警示：移植 social_state schema 但不补音频链路 = 表面跑通 Social 维永远空）。

### D5：可借鉴吸收（按 ROI）

- **Tier1 借鉴**：sample-size 分级 confidence 门（→ P0-2）、G1 RAG citation 范式（→ P0-3）
- **Tier2 借鉴**：K域 compute / R12 显式通路 / plugin-package-contract 直移植 / memory-host-sdk facade
- **不做**：memory-graph（@supermemory，语义鸿沟首屏必空白，V6 已有 starmap）、plugin-sdk 60 子路径（过度工程）、net-policy TS SSRF（V6 Python 版够用）
