# ADR-V6-001: Fork hermes-agent 为 V6 身体底座

- **Status**: Accepted
- **Date**: 2026-07-17
- **Supersedes**: (V5 自建 FastAPI 后端 —— V6 改用 fork 底座)

## Context
V6 绿场重建，产品目标不变（个人数据资产引擎），形态从手机 App → 电脑端客户端。需选「捕获层 + 使能器」成本最低的底座（V6 架构设计 §2.1：Hermes/OpenClaw 两底座都无数据大脑，大脑占 ~70% 新建工作量）。Hermes 胜出：6 个中国 messenger 通道全内置（生产级语音）/ Python 原生声学 / 168 技能 / Electron 纯回环 + SQLite / 单语言栈（2 人团队）。

## Decision
硬 fork `hermes-agent @ commit 0f102fa` → `realityos-desktop`。
- **License = MIT**（已核实），V6 可闭源商用，保留 `LICENSE` + 新建 `NOTICE` 注明来源。
- 保留 `upstream` remote（origin→upstream），选择性 cherry-pick hermes 更新。
- 换皮：`appId` com.realityos.desktop / `productName` RealityOS / 图标 / URL scheme。
- V6 ADR 新体系（ADR-V6-001 起），与 V5 的 ADR-001~093 分开存档。

## Consequences
- ✅ 继承：~5600 行 `conversation_loop`（不动）/ 6 通道 / `computer_use` 全谱系 / 168 技能 / MCP / cron / desktop shell。
- ✅ 身体白送，主战场集中在 brain（PTGProvider / Atomizer / PTG 11 表）。
- ⚠️ 移动靶：hermes 活跃提交，用「锁 conversation_loop 不动 + 选择性 merge + 记录基线 commit」控制。
- ⚠️ Nous 计费软耦合：stub 处理（见 ADR-V6-002 + Phase 0 P0-2）。
