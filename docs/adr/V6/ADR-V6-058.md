# ADR-V6-058 — 高风险技能 install 确认门（godmode 越狱工具双语风险提示）

**状态**: Accepted
**日期**: 2026-07-20
**关联**: [ADR-V6-053](ADR-V6-053.md) P1-T7 / 五份审计 T6 安全 / 铁律 C7 可观测 / [ADR-V6-037](ADR-V6-037.md) 反假绿

## 背景（审计发现 godmode 静默安装）

T6 安全审计（2026-07-20）发现：`optional-skills/security/godmode/` 是一个 **official 可选技能**，frontmatter 标记 `metadata.hermes.tags: [jailbreak, red-teaming, G0DM0D3, safety-bypass, uncensoring, ...]`，用途是「绕过 API 模型的安全过滤器」（含 auto_jailbreak + godmode_race）。

但 `hermes skills install godmode` 走的是 **official 分支**（`skills_hub.py:do_install` 确认门）——显示一个**良性的青色「Official Skill」面板**（"maintained by Nous Research, ships with hermes-agent"），与安装 `memory`/`search` 等普通技能**无差别**。

问题：
1. **静默降级**：用户没得到任何「这是越狱/红队工具」的信号，可能在不知情下安装到不拥有的模型上。
2. **C7 可观测缺口**：高风险双用途工具的安装无显著留痕/警示。
3. **official 信任被滥用**：godmode 因 source=official 自动获得 benign 面板，但它的 tags 明确标识为 safety-bypass——source 信任与内容风险是两回事。

## 决策

### D1：高危 tag/name 词表 + belt-and-suspenders 检测

`hermes_cli/skills_hub.py` 新增模块级常量与两个 helper：

```python
_HIGH_RISK_SKILL_TAGS = frozenset({
    "jailbreak", "red-teaming", "redteam", "safety-bypass",
    "uncensoring", "uncensored", "prompt-injection", "weight-tampering",
})
_HIGH_RISK_SKILL_NAMES = frozenset({"godmode", "obliteratus"})
```

`_collect_skill_tags(meta, extra_metadata, bundle_name)` 把 tag 信号**从所有可能落点**归一到一个 lowercase 集合：
- `meta.tags`（元数据模型字段）
- `extra_metadata["hermes"]["tags"]`（SKILL.md `metadata.hermes.tags` 块）
- `extra_metadata["tags"]`（扁平 tags）
- `bundle_name`（名字本身）

**为何 belt-and-suspenders**：上游解析器把 tags 放进哪个字段不完全可控（official/url/github 三路解析各异）；四路全收确保 gate **无论 tags 落在哪都触发**，不依赖单一字段被正确填充。

`_high_risk_skill_indicators(...)` 返回与高危词表的交集（空集 = 非高危）。

### D2：install 确认门加红色双语警告分支（优先级最高）

`do_install` 确认门原结构 `if official / else third-party` 改为 `if high-risk / elif official / else third-party`：

```python
_high_risk = _high_risk_skill_indicators(meta, extra_metadata, bundle.name)
if _high_risk:
    c.print(Panel( ... 红色双语 ⚠️ 高风险技能 / HIGH-RISK SKILL ... ,
                  border_style="red"))
elif bundle.source == "official":
    c.print(Panel( ... 青色 Official Skill ... ))
else:
    c.print(Panel( ... 黄色 third-party Disclaimer ... ))
```

红色面板内容（双语）：
- 标明 tags（如 `jailbreak, safety-bypass`）
- 明示「双用途安全研究工具，仅用于您拥有或已获明确授权的模型/系统」
- 明示「对不拥有/未授权模型使用可能违反 ToS 或当地法律，误用责任自负」
- 安装路径

**关键**：高危分支优先级**高于** source 分支——godmode 虽 source=official，但因 tags 命中，走红色门而非 benign 青色门。这把「source 信任」与「内容风险」解耦。

### D3：godmode SKILL.md 加双语风险提示节

`optional-skills/security/godmode/SKILL.md` 顶部（intro 之后）新增 `## ⚠️ Risk Warning / 风险提示` 节，双语陈述授权边界 + 误用责任 + 「install 时会弹红色确认面板」+ 「本技能不外泄数据、不持久化超出配置面，但设计目的就是绕过安全过滤器」。技能文档本身也承担告知义务，不只靠 install 门。

## 不做（诚实边界）

- **不**在 `skip_confirm`（TUI/gateway/slash 非交互面）上拒绝安装高危技能——那会破坏非交互 install 流，且超出「加警示」的审计目标。`skip_confirm` 本就是显式跳过确认的语义；非交互面的强阻断是更大的策略变更，本 ADR 不越界。**已安装技能的运行时影响不在本 ADR 范围**（godmode 一旦安装，其指令在 agent 会话中生效——这是技能安装的固有语义，非 install 门能管）。
- **不**自动扫描 SKILL.md 正文判定高危——只看结构化 tags + 已知名字。正文语义判定不可靠（NLP 误判），tags 是作者自声明，可信且可机器判。
- **不**把 godmode 从 official 移除——它是合法的红队研究技能；本 ADR 只补「安装前充分告知」，不禁用。
- **不**改 `force` 语义——`--force` 仍绕过所有确认门（显式用户覆盖）。

## 验证

- `tests/hermes_cli/test_adr_v6_058_high_risk_skill_gate.py`（C4，12 例）：
  - **`test_godmode_matched_by_name`**（**承重**）：godmode 即便无 tags 解析，靠**名字**命中 → gate 触发。直接反证审计怨点。
  - `test_obliteratus_matched_by_name` / 各高危 tag（jailbreak/safety-bypass/uncensoring/red-teaming/weight-tampering）命中。
  - `test_benign_skill_not_flagged`：普通技能（memory/productivity）**不**误报。
  - `test_case_insensitive` + `TestBeltAndSuspendersCollection`（4 例）：meta.tags / nested hermes.tags / flat tags / name 四路全收 + 去重 + 小写化。
  - `TestHighRiskSetsAreLocked`（2 例）：钉死词表不缩水。
- 回归：`test_skills_hub.py` + `test_skills_install_flags.py` + `test_skills_skip_confirm.py` 共 48 例零回归（gate 三分支结构未破坏既有 official/third-party 流）。
- 本地 ruff 净。

## 反假绿（本 ADR 自身的诚实性）

| 项 | 判定 | 依据 |
| --- | --- | --- |
| godmode 是否真被门挡（非只 print） | ✅ | `_high_risk_skill_indicators("godmode")` 非空 → 走红色分支（test_godmode_matched_by_name） |
| 是否仅 official 才警告 | ❌ 否 | 高危分支优先级高于 source，third-party 高危也走红门 |
| tags 落点是否单点依赖 | ❌ 否 | belt-and-suspenders 四路全收（TestBeltAndSuspendersCollection） |
| 是否禁用了 godmode | ❌ 未禁用 | 合法红队技能；只补告知，D3 不做 |
| 是否声称管了安装后运行时影响 | ❌ 不声称 | 安装后指令生效是固有语义；本 ADR 范围=install 门 |
| 是否覆盖 skip_confirm 非交互面 | ❌ 不覆盖 | 诚实边界：不破坏非交互流，D 不做 |
