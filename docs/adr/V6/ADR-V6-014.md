## 决策

### P0-1：quality_metrics 表（SCHEMA_VERSION 4→4 落地，v4 注释块）

§9 #8「质量度量基线」要求把提取质量（P/R/F1 + per-type）落库，供 §11 门禁回归追踪。已在 `plugins/memory/ptg/schema.py` 加 `quality_metrics` 表（metric_date / metric_type / numerator / denominator / value / meta_json + user_id/version/deleted_at），纳入 `C2_USER_TABLES` + `_RECONCILE_COLUMNS`。**已做。**

### P0-2：backup 调度 C4 回归（6 测试）

ADR-V6-012 的灾备面（backup + verify_backup + restore_drill）落了表与逻辑，但调度时序的边界（grace、并发、verify 失败不静默）缺回归。补 6 个测试锁定契约。**已做。**

### P0-3：relations 终态列（§9 #1/#5 不可逆投资）

§9 #1/#5 要求 relations 预留终态列（terminal-state），避免事后 ALTER 在生产大表上重写。已在 relations DDL 预置终态列，含 `_RECONCILE_COLUMNS` 增量 ALTER 兼容旧库。**已做。**

### P0-4：DPIA + 数据分级文档（Phase 0 PIPL 合规）

§55/§56 要求 DPIA，§13 要求合法性基础留痕。新增两份文档（非代码，但 C1 决策留痕）：

- `docs/security/data-classification.md` — 14 张 PTG 表逐字段 L0–L3 分级 + §13 合法性基础 + §5 出境流 + 控制矩阵（✅/🟡/❌）。
- `docs/security/dpia.md` — §55 触发分析 + §56 风险登记册（R-EGRESS 高未闭环 / R-LLM-EXFIL 中高 / R-DEVICE-LOSS 低已闭环 / R-DELETE 中未闭环 / R-MINOR 条件性）+ §24 自动化决策 + §31 未成年人 + §38 出境 + §57 事件响应。

**已做（文档）。** 风险登记册诚实标注 R-EGRESS/R-DELETE「未闭环」——闭环动作正是 P1-2/P1-3，本批次同期落地。

### P1-1：tool_events 捕获面（SCHEMA_VERSION 4→5，§9 #4 不可逆投资）

§9 #4 要求 tool_events 捕获面预留——post_tool_call 钩子此前仅写 audit-log（PHASE 0 占位），未沉淀到结构化表。升级到 v5：

- `schema.py` 加 `tool_events` 表（tool_name/tool_args/result_summary/status CHECK(^(ok|error)$/error_type/error_msg/duration_ms/extracted_via default 'post_tool_call'/quark_evidence default '[]'/llm_call_id/captured_at + 3 索引），纳入 C2 + reconcile。
- 新增 `plugins/memory/ptg/capture_schemas.py` 的 `CaptureEvent` pydantic 模型 + `from_hook_kwargs` 工厂（MAX_TOOL_ARGS_CHARS=4096 / MAX_RESULT_SUMMARY_CHARS=2048，PIPL §6 最小化）。**这是 C5-adjacent 闸门**：钩子产物必须过 schema 才入库，违例走 DLQ（含最小诊断，不 dump 全量 args）。
- `store.py` 加 `insert_tool_event`（C7 fail-open INSERT）+ `recent_tool_events`（newest-first，尊重 soft-delete）。
- `plugins/observability/ptg_capture/__init__.py` 的 `_on_post_tool_call` 从 log-only 升级为：resolve store+user → CaptureEvent.from_hook_kwargs（C5 闸门）→ 违例入 DLQ → insert_tool_event。全链 fail-open。

**已做（含接线）。** post_tool_call 不再是审计日志占位。

### P1-2：realityos_security 出网策略（§6.5 复用 / §6.6 新增出口白名单）

§6.5 net-policy 要求 SSRF/IP 段防御——**经核实 Hermes 已有 `tools/url_safety.is_safe_url`（继承自上游，非重复造轮子）**。V6 在其上叠加 §6.6 的 LLM 出口白名单层。新增插件 `plugins/realityos_security/`：

- `policy.py`：`LLM_PROVIDER_ALLOWLIST`（deepseek/bigmodel/openai/anthropic/x.ai 等真实 LLM host）+ `classify_url`（CAT_LLM_PROVIDER/CAT_TOOL_FETCH/CAT_BLOCKED_INTERNAL/CAT_UNKNOWN，复用 is_safe_url，原语不可用时 fail-CLOSED）+ `fetch_guard`（blocked_internal 永不被 toggle 旁路）+ `assert_llm_egress`（出口白名单断言）。
- `__init__.py` + `plugin.yaml` 导出与清单。

**已做。** 诚实标注：`fetch_guard`/`assert_llm_egress` 的**调用点接线**（即真正在 LLM/工具出口前调用）是桌面 UI 集成步骤，当前为可用模块待接线——非骨架，是「层已就位、调用点在后续 UI 批次」。

### P1-3：realityos_sovereignty 主权层（§6 四行使面 + 唯一合法 hard-DELETE）

§6 要求用户对自有图谱的四行使面。新增插件 `plugins/realityos_sovereignty/sovereignty.py`：

1. `cascade_soft_delete(mode='A'|'B', since=, until=)` — §6.2 双模删除。Mode A=空间回收（仅 memos，**永不触 PTG 原子**）；Mode B=完全遗忘（memos + 四 R-atom 事件表 + entities + relations 级联）。Phase 1 仅做 §6.2 阶段1（立即 soft-mark）。
2. `purge_soft_deleted(older_than_days=)` — §6.2 阶段2 物理清除。**这是 V6 唯一合法 hard-DELETE 面**，C2 合规因每条被删行均已 soft-delete 且过宽限期，§6.2 明文要求。opt-in，测试从不自动跑。
3. `export_user_data` / `export_user_data_json` — §6.8 一键 JSON 导出（<5s，PIPL §45 可携带）。逐表单遍无 join 无 LLM。
4. `set_minor_mode` / `is_minor` — §6.7 未成年人年龄闸（ptg_meta key）。on 时 Atomizer 须跳 R1/R9 生物原子 + 聚合画像。
5. `set_consent_tag` / `get_consent_summary` — §6.1 relations.consent_tag 行使面（local_only/shareable/restricted）。

**关键 C2 细节**：`realityos_users` 用 `id`（它即租户表），其余 C2 表用 `user_id`——`_USER_ID_COL` 映射 + 测试实证捕获（C4 价值兑现）。所有入口 fail-open（C7），返回结果 dict 而非抛异常。

**已做（含接线到 store 层）。** 诚实标注：调用这些原语的**桌面 UI**（删除对话框/导出按钮/年龄闸/同意开关）+ §6.2 阶段2 夜间 cron 是下一步——原语已就位且对真实 PTGStore 测试通过（16 测试，含 3000 memos 导出 <5s、宽限期 purge、consent_tag 往返）。

### P1-4：Honcho 休眠对外口径 + README 广告纠正（ADR-V6-006）

ADR-V6-006 决策 Honcho + 7 云记忆 provider = 休眠（代码保留不加载，max-1-external gate 强制 PTGProvider 为唯一 active）。但 4 份 README 仍带 `<a href="...honcho">Honcho</a>` 广告——对外口径与决策矛盾。已改 4 语言 README（en/zh/es/ur-pk），移除 Honcho 链，替换为 `<strong>本地人生图谱记忆（PTG）——数据不出设备。</strong>`。全仓 grep 确认 README* 零 Honcho 残留。**已做。**
